# 9.1 FAISS

## 1 技术简介

FAISS（Facebook AI Similarity Search）是Facebook开源的向量相似度搜索库，专门针对大规模向量数据的高效相似度检索而设计。FAISS提供了多种索引结构和搜索算法，能够在数十亿向量规模的数据集上实现毫秒级的最近邻搜索，是构建向量数据库和RAG系统的核心技术组件。该库使用C++实现，提供了Python接口，具有高性能、低内存占用的特点。

## 2 技术背景

随着深度学习的发展，向量嵌入技术被广泛应用于自然语言处理、图像检索、推荐系统等领域。然而，高维向量的精确搜索面临"维度诅咒"问题——随着维度增加，计算复杂度呈指数增长。传统的暴力搜索（Brute-force）在海量数据场景下根本无法满足实时性需求。FAISS的出现解决了这一难题，它通过精心设计的索引结构和近似最近邻（ANN）算法，在精度和速度之间取得了优秀的平衡，使得大规模向量检索成为可能。

## 3 核心概念

**向量索引（Index）**：FAISS中用于加速搜索的数据结构，通过预先处理向量数据构建索引，搜索时无需遍历全部数据。FAISS支持多种索引类型，包括Flat（暴力搜索）、IVF（倒排文件）、HNSW（分层导航小世界图）、PQ（乘积量化）等。

**距离度量（Metric）**：衡量向量相似度的指标，FAISS支持L2距离（欧几里得距离）和内积（Inner Product）两种度量方式。L2距离适合大多数场景，归一化后的内积等价于余弦相似度。

**聚类中心（Centroid）**：IVF系列索引使用K-means算法将向量空间划分为多个聚类，每个聚类有一个中心点，搜索时只需查找与查询向量最近的几个聚类。

**码本（Codebook）**：PQ索引中用于向量量化的参考集合，将高维向量压缩为低维编码，大幅降低存储成本和搜索计算量。

## 4 工作原理

FAISS的工作原理可以概括为"建索引"和"搜索"两个阶段。在建索引阶段，FAISS首先接收原始向量数据，根据选择的索引类型进行预处理。对于IVF索引，系统使用K-means算法将向量聚类到若干个簇中，并构建倒排索引结构；对于HNSW索引，系统构建分层图结构，上层稀疏下层稠密；对于PQ索引，系统将高维向量分割为多个子向量并进行量化编码。

在搜索阶段，系统首先将查询向量转换为目标域（如归一化处理），然后根据索引类型执行相应的搜索策略。Flat索引执行精确的暴力搜索；IVF索引首先定位最近的几个聚类中心，然后在这些聚类中搜索；HNSW索引从顶层开始，通过图链接逐步向下导航；PQ索引使用编码进行快速距离估算。搜索结果返回与查询向量最相似的K个向量的索引和距离。

## 5 技术实现

FAISS的技术实现主要包含以下几个关键方面。首先是索引创建，通过实例化不同的Index类（如IndexFlatL2、IndexIVFFlat、IndexHNSWFlat、IndexPQ等）来创建不同类型的索引，大部分索引在添加向量前需要进行训练（train）以构建内部数据结构。其次是向量添加，使用index.add(xb)方法将向量添加到索引中，向量必须转换为float32类型。第三是参数调优，不同索引类型有不同的参数需要调整，如IVF的nlist（聚类数）、nprobe（搜索的聚类数），HNSW的M（连接数）、efSearch（搜索宽度），PQ的m（子向量数）、nbits（编码位数）等。第四是搜索执行，使用index.search(xq, k)方法执行搜索，返回距离矩阵D和索引矩阵I。

## 6 安装与环境配置

### 6.1 系统要求

FAISS对运行环境的要求相对较低，但在不同操作系统上的安装方式有所差异。硬件方面，建议至少8GB内存，因为索引构建过程中需要额外的内存空间。CPU方面，FAISS支持多线程加速，建议使用多核CPU以获得更好的性能。操作系统方面，FAISS支持Linux、macOS和Windows系统。

### 6.2 安装方法

FAISS提供多种安装方式，满足不同场景的需求。以下是推荐的安装方法：

```bash
# 方法1：使用pip安装（推荐）
pip install faiss-cpu  # CPU版本

# 如果需要GPU支持
pip install faiss-gpu  # GPU版本

# 方法2：使用conda安装
conda install faiss-cpu -c conda-forge

# 方法3：从源码编译（需要完整功能）
git clone https://github.com/facebookresearch/faiss.git
cd faiss
./configure --prefix=/usr/local
make -j4
make install
```


## 7 Flat索引详解

### 7.1 IndexFlatL2原理

IndexFlatL2是最基础的索引类型，执行精确的暴力搜索（Brute-force Search），计算查询向量与数据库中所有向量的L2距离，然后返回距离最小的K个向量。这种索引的优点是搜索结果精确（100%召回率），缺点是搜索时间与数据规模线性相关，不适合大规模数据。

```python
import faiss
import numpy as np
import time

# ============ Flat索引完整示例 ============

# 参数设置
d = 128              # 向量维度
nb = 100000          # 数据库向量数量（10万）
nq = 100             # 查询向量数量
k = 10               # 返回top-k结果

# 生成测试数据
np.random.seed(42)
xb = np.random.random((nb, d)).astype('float32')
xq = np.random.random((nq, d)).astype('float32')

# 创建IndexFlatL2索引
index = faiss.IndexFlatL2(d)

print("=" * 60)
print("IndexFlatL2 索引演示")
print("=" * 60)
print(f"向量维度: {d}")
print(f"数据库规模: {nb}")
print(f"查询数量: {nq}")
print(f"返回结果数: {k}")

# 添加向量
start_time = time.time()
index.add(xb)
add_time = time.time() - start_time
print(f"\n添加 {index.ntotal} 个向量耗时: {add_time:.4f}秒")

# 执行搜索并计时
start_time = time.time()
D, I = index.search(xq, k)
search_time = time.time() - start_time

print(f"搜索耗时: {search_time:.4f}秒")
print(f"平均单次查询耗时: {search_time/nq*1000:.2f}ms")
print(f"吞吐量: {nq/search_time:.2f} QPS")

# 展示第一个查询的结果
print("\n第一个查询的top-{}结果:".format(k))
print("索引:", I[0])
print("距离:", D[0])
```

### 7.2 IndexFlatIP（内积索引）

IndexFlatIP使用内积作为距离度量，适用于余弦相似度搜索场景。使用时需要先将向量归一化，这样内积就等价于余弦相似度。

```python
import faiss
import numpy as np

# ============ IndexFlatIP（内积索引）示例 ============

d = 64                     # 向量维度
nb = 1000                  # 向量数量

# 生成归一化的测试向量
np.random.seed(123)
xb = np.random.random((nb, d)).astype('float32')

# 归一化向量（使内积等价于余弦相似度）
norms = np.linalg.norm(xb, axis=1, keepdims=True)
xb_normalized = xb / norms

# 创建内积索引
index = faiss.IndexFlatIP(d)

# 添加归一化后的向量
index.add(xb_normalized)

print(f"索引类型: IndexFlatIP (内积)")
print(f"向量数量: {index.ntotal}")

# 查询向量
xq = np.random.random((1, d)).astype('float32')
xq_normalized = xq / np.linalg.norm(xq)

# 搜索（返回的"距离"实际是内积值，越大越相似）
k = 5
D, I = index.search(xq_normalized, k)

print(f"\n查询向量的top-{k}相似结果:")
for i in range(k):
    print(f"  排名{i+1}: 索引={I[0][i]}, 内积(相似度)={D[0][i]:.4f}")
```

### 7.3 Batch处理优化

FAISS支持批量处理多个查询，可以显著提高吞吐量。下面的示例展示如何高效处理批量查询：

```python
import faiss
import numpy as np
import time

# ============ 批量搜索优化示例 ============

d = 128
nb = 50000
nq_list = [1, 10, 100, 1000]
k = 10

# 创建索引并添加数据
np.random.seed(42)
xb = np.random.random((nb, d)).astype('float32')
index = faiss.IndexFlatL2(d)
index.add(xb)

print("=" * 60)
print("批量搜索性能测试")
print("=" * 60)

for nq in nq_list:
    xq = np.random.random((nq, d)).astype('float32')
    
    # 多次测试取平均
    times = []
    for _ in range(5):
        start = time.time()
        D, I = index.search(xq, k)
        times.append(time.time() - start)
    
    avg_time = np.mean(times)
    qps = nq / avg_time
    
    print(f"查询数: {nq:4d} | "
          f"平均耗时: {avg_time*1000:8.2f}ms | "
          f"QPS: {qps:8.2f}")
```

## 8 IVF索引详解

### 8.1 IVF基本原理

IVF（Inverted File倒排文件）索引通过聚类将数据划分为多个桶（Cluster），搜索时只查询与查询向量最近的几个桶，从而减少搜索范围，提高搜索速度。这种方法称为"近似最近邻搜索"（ANN），牺牲一定的召回率换取速度提升。

IVF索引的核心参数包括：
- `nlist`: 聚类数量，通常设置为数据量的平方根左右
- `nprobe`: 搜索时查询的聚类数量，越多召回率越高但越慢

```python
import faiss
import numpy as np
import time

# ============ IVF索引完整示例 ============

# 参数配置
d = 128                      # 向量维度
nb = 100000                  # 数据库规模（10万）
nq = 100                     # 查询数量
k = 10                       # 返回top-k
nlist = 100                  # 聚类数量
nprobe = 10                  # 查询的聚类数量

# 生成测试数据
np.random.seed(42)
xb = np.random.random((nb, d)).astype('float32')
xq = np.random.random((nq, d)).astype('float32')

print("=" * 60)
print("IVF索引演示")
print("=" * 60)

# 方法1：使用IVFFlat（精确但内存占用较大）
# 第一步：创建量化器（用于计算聚类中心）
quantizer = faiss.IndexFlatL2(d)

# 第二步：创建IVFFlat索引
# 参数：量化器、维度、聚类数量
index_ivf = faiss.IndexIVFFlat(quantizer, d, nlist, faiss.METRIC_L2)

print(f"索引创建前训练状态: {index_ivf.is_trained}")

# 第三步：训练索引（建立聚类中心）
# 训练数据应该是数据向量的一个子集，通常使用全部数据
start_time = time.time()
index_ivf.train(xb)
train_time = time.time() - start_time
print(f"训练耗时: {train_time:.2f}秒")
print(f"训练后状态: {index_ivf.is_trained}")

# 第四步：添加向量
start_time = time.time()
index_ivf.add(xb)
add_time = time.time() - start_time
print(f"添加 {index_ivf.ntotal} 个向量耗时: {add_time:.2f}秒")

# 第五步：设置搜索参数
index_ivf.nprobe = nprobe

# 第六步：执行搜索
start_time = time.time()
D, I = index_search(xq, k)
search_time = time.time() - start_time

print(f"\n搜索参数: nlist={nlist}, nprobe={nprobe}")
print(f"搜索耗时: {search_time:.4f}秒")
print(f"平均单次查询: {search_time/nq*1000:.2f}ms")
```

### 8.2 IVF-PQ复合索引

IVF-PQ结合了IVF和PQ的优点，进一步压缩存储空间并提高搜索速度。PQ（Product Quantization乘积量化）将高维向量分割为多个子空间，分别进行量化，从而大幅降低存储需求。

```python
import faiss
import numpy as np
import time

# ============ IVF-PQ索引示例 ============

# 参数配置
d = 128                      # 向量维度
nb = 100000                  # 数据库规模
nq = 100                     # 查询数量
k = 10                       # 返回top-k
nlist = 100                  # IVF聚类数量
m = 8                        # PQ子向量数量（维度被分割为m份）
nbits = 8                    # 每个子向量的编码位数

# 生成测试数据
np.random.seed(42)
xb = np.random.random((nb, d)).astype('float32')
xq = np.random.random((nq, d)).astype('float32')

print("=" * 60)
print("IVF-PQ索引演示（高压缩率）")
print("=" * 60)

# 创建量化器
quantizer = faiss.IndexFlatL2(d)

# 创建IVF-PQ索引
# 参数：量化器、维度、聚类数、子向量数、编码位数、距离度量
index_pq = faiss.IndexIVFPQ(
    quantizer, 
    d, 
    nlist, 
    m, 
    nbits, 
    faiss.METRIC_L2
)

# 训练索引
print("开始训练...")
start_time = time.time()
index_pq.train(xb)
print(f"训练耗时: {time.time() - start_time:.2f}秒")

# 添加向量
start_time = time.time()
index_pq.add(xb)
add_time = time.time() - start_time
print(f"添加 {index_pq.ntotal} 个向量耗时: {add_time:.2f}秒")

# 设置搜索参数
index_pq.nprobe = 10

# 搜索
start_time = time.time()
D, I = index_pq.search(xq, k)
search_time = time.time() - start_time

print(f"搜索耗时: {search_time:.4f}秒")
print(f"平均单次查询: {search_time/nq*1000:.2f}ms")
print(f"索引占用的内存（估算）: {index_pq.ntotal * m * nbits / 8 / 1024 / 1024:.2f} MB")

# 对比：纯内存占用
flat_index = faiss.IndexFlatL2(d)
flat_index.add(xb)
flat_size = nb * d * 4 / 1024 / 1024  # float32
print(f"Flat索引内存占用（估算）: {flat_size:.2f} MB")
print(f"压缩比: {flat_size / (index_pq.ntotal * m * nbits / 8 / 1024 / 1024):.1f}x")
```

### 8.3 召回率调优

IVF索引需要在速度和召回率之间权衡。以下示例展示如何调整参数以获得最佳召回率：

```python
import faiss
import numpy as np

# ============ IVF召回率调优示例 ============

d = 128
nb = 50000
nq = 100
k = 10

# 生成测试数据
np.random.seed(42)
xb = np.random.random((nb, d)).astype('float32')
xq = np.random.random((nq, d)).astype('float32')

# 创建基准索引（Flat，用于计算真实最近邻）
index_flat = faiss.IndexFlatL2(d)
index_flat.add(xb)
D_flat, I_flat = index_flat.search(xq, k)

# 创建IVF索引
quantizer = faiss.IndexFlatL2(d)
nlist = 100
index_ivf = faiss.IndexIVFFlat(quantizer, d, nlist)
index_ivf.train(xb)
index_ivf.add(xb)

print("=" * 60)
print("nprobe参数与召回率关系")
print("=" * 60)
print(f"{'nprobe':<10} {'耗时(ms)':<15} {'召回率':<15}")
print("-" * 40)

for nprobe in [1, 2, 5, 10, 20, 50, 100]:
    index_ivf.nprobe = nprobe
    
    start = time.time()
    D_ivf, I_ivf = index_ivf.search(xq, k)
    elapsed = time.time() - start
    
    # 计算召回率
    recall = 0
    for i in range(nq):
        true_set = set(I_flat[i])
        result_set = set(I_ivf[i])
        recall += len(true_set & result_set) / k
    recall /= nq
    
    print(f"{nprobe:<10} {elapsed*1000/nq:<15.2f} {recall*100:<15.2f}%")
```

## 9 HNSW索引详解

### 9.1 HNSW原理

HNSW（Hierarchical Navigable Small World Graph）是一种基于图的近似最近邻搜索算法。它的核心思想是构建一个分层图结构：上层节点稀疏，连接距离较远；下层节点密集，连接距离较近。搜索时从上层开始，通过贪心算法逐步向下导航，最终找到最近邻。

HNSW的主要参数：
- `M`: 每个节点的连接数（边的数量），默认32
- `efConstruction`: 建索引时的搜索宽度，默认40
- `efSearch`: 搜索时的搜索宽度，默认16

```python
import faiss
import numpy as np
import time

# ============ HNSW索引完整示例 ============

# 参数配置
d = 128                      # 向量维度
nb = 100000                  # 数据库规模（10万）
nq = 100                     # 查询数量
k = 10                       # 返回top-k
M = 32                       # 每个节点的连接数
efConstruction = 40          # 建索引时的搜索宽度
efSearch = 16                # 搜索时的搜索宽度

# 生成测试数据
np.random.seed(42)
xb = np.random.random((nb, d)).astype('float32')
xq = np.random.random((nq, d)).astype('float32')

print("=" * 60)
print("HNSW索引演示")
print("=" * 60)

# 创建HNSW索引
# 参数：维度、每个节点的连接数(M)、efConstruction
index_hnsw = faiss.IndexHNSWFlat(d, M)
index_hnsw.hnsw.efConstruction = efConstruction
index_hnsw.hnsw.efSearch = efSearch

print(f"参数配置: M={M}, efConstruction={efConstruction}, efSearch={efSearch}")

# 添加向量
start_time = time.time()
index_hnsw.add(xb)
add_time = time.time() - start_time
print(f"添加 {index_hnsw.ntotal} 个向量耗时: {add_time:.2f}秒")

# 执行搜索
start_time = time.time()
D, I = index_hnsw.search(xq, k)
search_time = time.time() - start_time

print(f"\n搜索耗时: {search_time:.4f}秒")
print(f"平均单次查询: {search_time/nq*1000:.2f}ms")
print(f"吞吐量: {nq/search_time:.2f} QPS")

# 展示第一个查询结果
print(f"\n第一个查询的top-{k}结果:")
print(f"索引: {I[0]}")
print(f"距离: {D[0]}")
```

### 9.2 HNSW参数调优

HNSW的参数对搜索性能和召回率有显著影响。以下示例展示不同参数配置的效果：

```python
import faiss
import numpy as np
import time

# ============ HNSW参数调优示例 ============

d = 128
nb = 50000
nq = 50
```

### 10.1 完整搜索流程示例

以下是一个完整的向量搜索流程示例，涵盖从数据准备到结果返回的全过程：

```python
import faiss
import numpy as np
import time
from typing import List, Tuple

class VectorSearchEngine:
    """向量搜索引擎封装类"""
    
    def __init__(self, d: int, index_type: str = "flat"):
        """
        初始化搜索引擎
        
        参数:
            d: 向量维度
            index_type: 索引类型 ('flat', 'ivf', 'hnsw', 'ivf_pq')
        """
        self.d = d
        self.index_type = index_type
        self.index = None
        
    def build_index(self, vectors: np.ndarray, nlist: int = 100, 
                    M: int = 32, efSearch: int = 16):
        """
        构建索引
        
        参数:
            vectors: 原始向量数据，形状为 (n, d)
            nlist: IVF聚类数量
            M: HNSW连接数
            efSearch: HNSW搜索宽度
        """
        vectors = self._preprocess(vectors)
        
        if self.index_type == "flat":
            self.index = faiss.IndexFlatL2(self.d)
            self.index.add(vectors)
            
        elif self.index_type == "ivf":
            quantizer = faiss.IndexFlatL2(self.d)
            self.index = faiss.IndexIVFFlat(quantizer, self.d, nlist)
            self.index.train(vectors)
            self.index.add(vectors)
            
        elif self.index_type == "hnsw":
            self.index = faiss.IndexHNSWFlat(self.d, M)
            self.index.hnsw.efSearch = efSearch
            self.index.add(vectors)
            
        elif self.index_type == "ivf_pq":
            quantizer = faiss.IndexFlatL2(self.d)
            m = 8  # PQ子向量数
            nbits = 8
            self.index = faiss.IndexIVFPQ(quantizer, self.d, nlist, m, nbits)
            self.index.train(vectors)
            self.index.add(vectors)
            
        else:
            raise ValueError(f"不支持的索引类型: {self.index_type}")
            
        return self
        
    def search(self, query: np.ndarray, k: int = 10) -> Tuple[np.ndarray, np.ndarray]:
        """
        搜索最近邻
        
        参数:
            query: 查询向量，形状为 (nq, d) 或 (d,)
            k: 返回的最近邻数量
            
        返回:
            (distances, indices): 距离矩阵和索引矩阵
        """
        if query.ndim == 1:
            query = query.reshape(1, -1)
        query = self._preprocess(query)
        
        return self.index.search(query, k)
    
    def set_search_params(self, **kwargs):
        """设置搜索参数"""
        if self.index_type == "ivf" or self.index_type == "ivf_pq":
            if 'nprobe' in kwargs:
                self.index.nprobe = kwargs['nprobe']
        elif self.index_type == "hnsw":
            if 'efSearch' in kwargs:
                self.index.hnsw.efSearch = kwargs['efSearch']
    
    def _preprocess(self, vectors: np.ndarray) -> np.ndarray:
        """预处理向量"""
        return vectors.astype('float32')
    
    def get_vector_count(self) -> int:
        """获取索引中的向量数量"""
        return self.index.ntotal if self.index else 0


# 使用示例
```

```python
import faiss
import numpy as np
from typing import Dict, List, Any
import json

class MetadataVectorStore:
    """带元数据的向量存储"""
    
    def __init__(self, d: int, index_type: str = "flat"):
        self.d = d
        self.index_type = index_type
        self.index = None
        self.metadata: List[Dict[str, Any]] = []
        self.id_to_idx: Dict[int, int] = {}
        
    def add_vectors(self, vectors: np.ndarray, metadata: List[Dict]):
        """
        添加向量和元数据
        
        参数:
            vectors: 向量数组 (n, d)
            metadata: 元数据列表，长度为n
        """
        vectors = vectors.astype('float32')
        
        if self.index is None:
            self._create_index(vectors.shape[0])
        
        # 构建索引
        start_idx = self.index.ntotal
        self.index.add(vectors)
        
        # 记录元数据
        for i, meta in enumerate(metadata):
            self.metadata.append(meta)
            # 假设元数据包含id字段
            if 'id' in meta:
                self.id_to_idx[meta['id']] = start_idx + i
    
    def _create_index(self, nb: int):
        """创建索引"""
        if self.index_type == "flat":
            self.index = faiss.IndexFlatL2(self.d)
        elif self.index_type == "hnsw":
            self.index = faiss.IndexHNSWFlat(self.d, 32)
        # 可以添加更多索引类型
    
    def search(self, query: np.ndarray, k: int = 10) -> List[Dict]:
        """
        搜索并返回带元数据的结果
        
        参数:
            query: 查询向量
            k: 返回数量
            
        返回:
            包含元数据的搜索结果列表

                    'rank': i + 1,
                    'distance': float(distances[0][i]),
                    'index': int(idx),
                    'metadata': self.metadata[idx]
                })
        
        return results
```

### 11.2 使用NumPy进行高效数据处理

```python
import faiss
import numpy as np
from typing import List, Tuple

class EfficientVectorProcessor:
    """高效的向量处理工具类"""
    
    @staticmethod
    def create_batches(data: np.ndarray, batch_size: int) -> List[np.ndarray]:
        """
        将大数据分批处理
        
        参数:
            data: 完整数据 (n, d)
            batch_size: 批量大小
            
        返回:
            分批后的数据列表
        """
        n = data.shape[0]
        batches = []
        for i in range(0, n, batch_size):
            batches.append(data[i:i+batch_size])
        return batches
    
    @staticmethod
    def batch_add_index(index, vectors: np.ndarray, batch_size: int = 10000):
        """
        分批添加向量到索引
        
        参数:
            index: FAISS索引
            vectors: 向量数据
            batch_size: 批量大小
        """
        batches = EfficientVectorProcessor.create_batches(vectors, batch_size)
        for batch in batches:
            index.add(batch)
    
    @staticmethod
    def batch_search(index, queries: np.ndarray, k: int, 
                     batch_size: int = 1000) -> Tuple[np.ndarray, np.ndarray]:
        """
        分批执行搜索
        
        参数:
            index: FAISS索引
            queries: 查询向量
            k: 返回数量
            batch_size: 批量大小
            
        返回:
            (distances, indices)
        """
        nq = queries.shape[0]
        all_distances = np.zeros((nq, k), dtype='float32')
        all_indices = np.full((nq, k), -1, dtype='int64')
        
        for i in range(0, nq, batch_size):
            batch_queries = queries[i:i+batch_size]
            D, I = index.search(batch_queries, k)
            
            end_idx = min(i + batch_size, nq)
            all_distances[i:end_idx] = D
            all_indices[i:end_idx] = I
        
        return all_distances, all_indices
```

## 12 常见问题

**问题一：向量维度不匹配**
FAISS要求所有向量具有相同的维度，在创建索引时需要指定维度参数d。添加向量时，如果维度不匹配会抛出异常。解决方案是在添加前确保所有向量维度一致。

**问题二：数据类型错误**
FAISS要求输入向量为float32类型，如果使用float64或其他类型会导致错误或性能下降。解决方案是使用.astype('float32')进行类型转换。

**问题三：索引未训练错误**
对于IVF、PQ等需要训练的索引类型，必须先调用train方法才能添加向量。Flat索引不需要训练，is_trained属性为True。

**问题四：内存占用过高**
HNSW等索引在构建时会消耗大量内存，特别是M参数设置较大时。可以通过减小M参数、使用IVF-PQ等方式降低内存占用。

**问题五：搜索召回率低**
近似最近邻索引在追求速度的同时会牺牲一定精度。可以通过调整参数（如HNSW增大efSearch，IVF增大nprobe）来提高召回率。

## 13 应用场景

**向量数据库构建**：FAISS是构建高性能向量数据库的核心组件，可以存储数十亿向量并支持毫秒级检索，广泛应用于知识库检索、语义搜索等场景。

**相似图像检索**：将图像通过CNN提取特征向量后，使用FAISS进行相似图像搜索，可用于图片去重、相似商品推荐、版权检测等应用。

**推荐系统**：将用户和物品表示为向量，使用FAISS快速找到与用户最相似的Top-K物品，实现个性化推荐。

**文本语义搜索**：将文档和查询转换为向量表示，通过FAISS检索最相关的文档，实现基于语义的全文搜索，显著提升搜索体验。

**RAG系统**：在检索增强生成系统中，使用FAISS作为向量索引，高效检索与用户问题相关的知识片段，提升生成质量。
