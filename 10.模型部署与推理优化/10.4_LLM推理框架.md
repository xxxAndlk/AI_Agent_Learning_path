# 10.4 LLM推理框架

## 1 技术简介

**技术定义**：LLM推理框架是用于部署和运行大语言模型的专业工具，提供高性能推理能力。这些框架通过KV Cache优化、算子融合、批处理等技术，显著提升模型推理吞吐量和降低延迟。与通用的深度学习推理引擎不同，LLM推理框架专门针对自回归语言模型进行优化，能够更高效地处理变长序列和批量请求。

随着大语言模型（LLM）的广泛应用，如何高效地部署和运行这些模型成为一个关键挑战。LLM具有以下特点：
- 自回归生成：逐token生成，需要缓存历史kv值
- 变长输入：不同请求的输入长度可能差异很大
- 内存密集：KV Cache占用大量显存
- 延迟敏感：用户对响应时间有较高期望

LLM推理框架正是为了解决这些问题而设计的专用工具。它们不仅提供高效的推理能力，还提供完整的API服务、量化支持、多GPU扩展等功能。

**主流LLM推理框架概述**：
- **vLLM**：开源、高性能、易用，PagedAttention技术
- **TensorRT-LLM**：NVIDIA官方，极致性能优化
- **TGI**：HuggingFace官方生产级推理服务器
- **Ollama**：简单易用，本地部署首选

## 2 技术背景

**LLM部署的挑战**：
传统的深度学习框架（如PyTorch）直接用于LLM推理存在以下问题：
- KV Cache管理低效：预分配固定长度的内存，大量浪费
- 批处理效率低：静态批处理，需要等待所有请求完成
- 内存碎片化：不同请求的序列长度差异导致内存碎片
- 无法充分利用硬件：缺少针对LLM的特殊优化

**技术发展历程**：
1. 早期使用PyTorch直接部署，功能正确但效率低
2. 引入TensorRT优化，但配置复杂
3. 2023年vLLM发布，带来PagedAttention创新
4. 各厂商纷纷推出自己的LLM推理框架

**框架选择的重要性**：
选择合适的推理框架可以：
- 提升2-10倍的吞吐量
- 降低50%以上的延迟
- 减少显存占用，支持更大batch size
- 简化部署和运维

## 3 核心概念

**PagedAttention（分页注意力）**：
vLLM提出的创新技术，借鉴操作系统的虚拟内存分页管理思想：
- 将KV Cache分成固定大小的页面进行管理
- 按需分配和释放内存页
- 物理页和虚拟页分离，减少碎片
- 显著提升显存利用率，支持更多并发

**Continuous Batching（连续批处理）**：
- 传统批处理需要等待batch中所有请求完成
- 连续批处理允许新请求随时加入
- 请求完成立即释放资源，新请求填补空位
- 提高GPU利用率，减少请求等待时间

**算子融合**：
- 将多个小算子合并为一个CUDA kernel
- 减少kernel launch开销和显存访问
- 典型融合：LayerNorm + Add + GeLU

**张量并行**：
- 将模型分布在多个GPU上
- 每GPU负责模型的一部分计算
- 支持更大的模型和更大的batch size

**量化支持**：
- INT8、INT4、FP8量化
- 减少显存占用，提升推理速度
- AWQ、GPTQ等量化方案

## 4 工作原理

**vLLM工作原理**：
1. 启动时加载模型到GPU显存
2. 为每个请求创建虚拟的KV Cache空间
3. 使用PagedAttention管理物理内存
4. 动态调度请求到GPU执行
5. 请求完成立即释放资源

**TensorRT-LLM工作原理**：
1. 将HuggingFace模型转换为TensorRT格式
2. 使用TensorRT编译器进行图优化
3. 应用算子融合、量化等优化
4. 生成优化的CUDA kernel
5. 运行时加载优化后的引擎执行推理

**TGI工作原理**：
1. 基于HuggingFace Transformers构建
2. 实现了连续的批处理调度器
3. 支持Flash Attention加速
4. 内置多种量化方案
5. 提供OpenAI兼容的API

**Ollama工作原理**：
1. 将模型下载到本地
2. 使用Go语言构建高效服务
3. 自动选择最优的运行参数
4. 提供REST API和CLI接口

## 5 技术实现

**vLLM实现要点**：
- 使用LLM类加载模型
- 配置SamplingParams控制生成
- 调用generate方法进行推理
- 支持OpenAI兼容API

**TensorRT-LLM实现步骤**：
1. 转换模型权重到TensorRT格式
2. 使用trtllm-build构建引擎
3. 编写推理代码加载引擎
4. 执行推理

**TGI实现方式**：
- 使用Docker镜像部署
- 通过HTTP API调用
- 支持流式输出

**Ollama使用流程**：
- 一行命令安装
- 拉取模型
- 运行推理

def vllm_basic_example():
    """4. vLLM基础推理示例
    
    展示vLLM的基本使用方法，包括模型加载、参数配置和批量推理
    """
    try:
        # 5. 导入vLLM核心模块
        from vllm import LLM, SamplingParams
        
        print("=" * 60)
        print("vLLM基础推理示例")
        print("=" * 60)
        
        # 6. 步骤1: 初始化LLM引擎
        # LLM类是vLLM的核心类，负责模型加载和推理管理
        llm = LLM(
            model="Qwen/Qwen2.5-0.5B-Instruct",  # 7. 模型名称，支持HuggingFace格式
            tensor_parallel_size=1,               # 8. 张量并行数，多GPU时设为GPU数量
            gpu_memory_utilization=0.9,           # 9. GPU显存利用率，0.9表示使用90%显存
            max_model_len=4096,                   # 10. 最大序列长度，影响KV Cache大小
            trust_remote_code=True,               # 11. 是否信任远程代码（部分模型需要）
            dtype="auto",                         # 12. 数据类型，auto自动选择最优
        )
        print("✅ 模型加载完成")
        
        # 13. 步骤2: 配置采样参数
        # SamplingParams控制文本生成的随机性和长度
        sampling_params = SamplingParams(
            temperature=0.7,                      # 14. 温度参数，控制随机性（0-2）
                                                    # 低温度(0.3)更确定，高温度(1.0+)更随机
            top_p=0.9,                            # 15. 核采样参数，保留累积概率为p的token
            top_k=50,                             # 16. Top-K采样，只从概率最高的K个token中采样
            max_tokens=256,                       # 17. 最大生成token数
            frequency_penalty=0.0,                # 18. 频率惩罚，减少重复词（0-2）
            presence_penalty=0.0,                 # 19. 存在惩罚，鼓励话题多样性（0-2）
            stop=["<|endoftext|>", "###"],       # 20. 停止词，遇到这些词时停止生成
        )
        
        # 21. 步骤3: 准备输入提示词
        # vLLM支持批量推理，可以一次处理多个请求
        prompts = [
            "请用一句话解释什么是机器学习：",
            "Python语言的主要特点是什么？",
            "深度学习和机器学习有什么区别？",
        ]
        
        # 22. 步骤4: 执行批量推理
        # vLLM会自动进行批处理优化，提高GPU利用率
        print("\n开始推理...")
        outputs = llm.generate(prompts, sampling_params)
        
        # 23. 步骤5: 处理并输出结果
        # outputs是一个RequestOutput对象列表
        for i, output in enumerate(outputs):
            print(f"\n--- 结果 {i+1} ---")
            print(f"输入: {output.prompt}")
            print(f"输出: {output.outputs[0].text}")
            print(f"生成token数: {len(output.outputs[0].token_ids)}")
        
        print("\n✅ 推理完成")
        
    except ImportError:
        print("⚠️ 请先安装vLLM: pip install vllm")
        print("注意: vLLM需要Linux环境和CUDA支持的GPU")

def vllm_streaming_example():
    """29. vLLM流式输出示例
    
    展示如何使用vLLM进行流式文本生成
    """
    try:
        import openai
        
        print("\n" + "=" * 60)
        print("vLLM流式输出示例")
        print("=" * 60)
        
        client = openai.OpenAI(
            base_url="http://localhost:8000/v1",
            api_key="dummy-key"
        )
        
        # 30. 发送流式请求
        # stream=True 启用流式输出
        stream = client.chat.completions.create(
            model="Qwen/Qwen2.5-0.5B-Instruct",
            messages=[
                {"role": "user", "content": "请写一首关于春天的短诗。"}
            ],
            temperature=0.8,
            max_tokens=128,
            stream=True,                           # 31. 启用流式输出
        )

        # 32. 逐块处理流式响应
        for chunk in stream:
            if chunk.choices[0].delta.content is not None:
                print(chunk.choices[0].delta.content, end="", flush=True)
        
        print("\n\n✅ 流式输出完成")
        
    except Exception as e:
        print(f"⚠️ 流式输出示例需要vLLM服务运行: {e}")

import tensorrt_llm.bindings as bindings # C++绑定
from tensorrt_llm.runtime import ModelConfig, SamplingConfig
import torch                             # PyTorch张量操作

class TRTLLMEngine:
    '''TensorRT-LLM推理引擎封装'''
    
    def __init__(self, engine_dir: str):
        '''8. 初始化TRT-LLM引擎
        
        参数:
            engine_dir: TRT引擎目录路径
        '''
        # 加载引擎配置
        config_path = os.path.join(engine_dir, "config.json")
        with open(config_path, 'r') as f:
            config = json.load(f)
        
        # 9. 初始化模型配置
        self.model_config = ModelConfig(
            max_batch_size=config["max_batch_size"],
            max_input_len=config["max_input_len"],
            max_output_len=config["max_output_len"],
        )
        
        # 10. 加载引擎
        print(f"加载引擎: {engine_dir}")
        self.engine = bindings.GptSession(
            engine_dir=engine_dir,
        )
        print("✅ 引擎加载完成")
    
    def generate(
        self,
        input_tokens: List[List[int]],
        max_new_tokens: int = 256,
        temperature: float = 0.7,
        top_p: float = 0.9,
    ) -> List[List[int]]:
        '''11. 执行批量推理
        
        参数:
            input_tokens: 输入token ID列表（批量）
            max_new_tokens: 最大生成token数
            temperature: 采样温度
            top_p: 核采样参数
        返回:

            temperature=temperature,
            top_p=top_p,
            max_new_tokens=max_new_tokens,
        )
        
        # 13. 转换为张量格式
        input_ids = torch.tensor(input_tokens, dtype=torch.int32)
        
        # 14. 执行推理
        with torch.no_grad():
            output_ids = self.engine.generate(
                input_ids=input_ids,
                sampling_config=sampling_config,
            )
        
        return output_ids.tolist()

## 7 常见问题

**安装问题**：
- vLLM/TensorRT-LLM需要Linux环境和NVIDIA GPU
- CUDA版本需要与框架兼容
- Docker环境配置复杂

**性能问题**：
- 未启用硬件加速
- 批量大小设置不当
- 未使用量化

**模型加载失败**：
- 模型格式不兼容
- 显存不足
- 模型路径错误

**API调用问题**：
- 服务未正确启动
- 端口配置错误
- 认证问题

## 8 应用场景

**本地开发测试**：
- 使用Ollama快速验证模型效果
- 本地 prototyping 和调试

**生产服务部署**：
- vLLM/TensorRT-LLM/TGI用于高并发服务
- 支持OpenAI兼容API

**成本敏感场景**：
- 使用vLLM降低成本
- 使用量化减少资源需求

**极致性能场景**：
- TensorRT-LLM用于超低延迟需求
- 针对特定硬件优化

**HuggingFace生态**：
- TGI用于HuggingFace模型部署
- 兼容Transformers生态
