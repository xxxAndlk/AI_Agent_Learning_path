# 6.6 Agent Skills系统

## 1 技术简介

Agent Skills系统是Agent（智能体）的能力抽象层，它将各种能力封装为可复用、可组合的"技能"模块。每个技能是一个独立的、自包含的功能单元，Agent可以动态加载、组合和执行多个技能来实现复杂任务。

### Skills与Tools的区别

在Agent架构中，Tools（工具）和Skills（技能）是两个不同的概念，它们在粒度、状态管理、组合方式等方面存在显著差异：

| 维度 | Tools（工具） | Skills（技能） |
|------|-------------|--------------|
| **粒度** | 细粒度函数调用，如"调用API"、"发送邮件" | 粗粒度能力封装，如"数据分析"、"代码生成" |
| **状态** | 通常无状态，每次调用独立 | 可有状态，维护内部状态（如记忆、上下文） |
| **组合** | 简单调用，线性执行 | 可编排组合，支持条件分支、循环等复杂流程 |
| **学习** | 固定实现，不具备学习能力 | 支持学习和优化，可根据经验改进 |
| **复用** | 函数级复用 | 组件级复用，整体能力复用 |

### 核心作用

Skills系统的核心作用体现在以下几个方面：

1. **能力抽象**：将复杂能力封装为标准化组件，简化Agent开发
2. **动态组合**：支持多个技能按需组合，应对不同任务场景
3. **可扩展性**：新增技能只需实现标准接口，不影响现有系统
4. **状态管理**：技能可维护自身状态，实现复杂业务逻辑
5. **复用提升**：一次开发多处使用，提高开发效率

---

## 2 技术背景

### 为什么会出现Skills系统

在Agent发展的早期阶段，Agent主要通过调用各种Tools（工具）来完成任务。这种方式存在以下问题：

1. **能力粒度过细**：每个Tool只完成单一具体动作，Agent需要大量Tool配合才能完成复杂任务
2. **状态管理困难**：Tools通常无状态，复杂任务的中间状态难以维护
3. **编排复杂度高**：多个Tool的调用顺序、条件判断、错误处理等逻辑复杂
4. **复用性差**：相同的能力组合需要重复编写

### Skills系统解决了什么问题

Skills系统的出现解决了以下核心问题：

1. **封装复杂能力**：将多个相关操作封装为统一的能力单元，如"数据分析技能"包含数据获取、清洗、分析、可视化等一系列操作

2. **状态内聚**：技能可以维护自身状态，例如MemorySkill维护短期和长期记忆，ReflectionSkill维护反思历史

3. **简化编排**：通过技能组合（Skill Composition）而非Tool编排来实现复杂流程，降低开发复杂度

4. **支持学习**：技能可以在执行过程中积累经验，进行自我优化

5. **分层设计**：区分Core Skills（核心技能）、Domain Skills（领域技能）、Meta Skills（元技能），实现关注点分离

## 3 核心概念

### 什么是Skill（技能）

Skill（技能）是Agent能力的基本单元，它是一个具有以下特征的功能组件：

- **自包含**：技能封装了完成特定功能所需的所有逻辑和数据
- **可复用**：同一技能可在不同场景下重复使用
- **可组合**：多个技能可以组合形成更复杂的能力
- **可扩展**：新增技能不影响现有系统

### 技能类型

#### 3.1 Core Skills（核心技能）

核心技能是Agent运行所必需的基础能力，主要包括：

- **Memory Skill（记忆技能）**：管理短期和长期记忆，包括信息存储、检索、遗忘等
- **Planning Skill（规划技能）**：将复杂任务分解为可执行的子任务，制定执行计划
- **Reflection Skill（反思技能）**：评估执行结果，提取经验教训，改进未来表现

#### 3.2 Domain Skills（领域技能）

领域技能是针对特定应用领域的能力封装：

- **Code Generation Skill（代码生成技能）**：根据需求描述生成代码
- **Data Analysis Skill（数据分析技能）**：对数据进行统计分析，生成报告
- **Web Search Skill（网络搜索技能）**：搜索网络信息，获取外部知识
- **Document Writing Skill（文档写作技能）**：根据要求撰写各类文档

#### 3.3 Meta Skills（元技能）

元技能是关于技能本身的能力，使Agent具备"学习如何学习"的能力：

- **Skill Learning Skill（学习技能技能）**：学习掌握新技能的能力
- **Skill Composition Skill（组合技能技能）**：将多个技能组合为新技能的能力
- **Skill Evaluation Skill（评估技能技能）**：评估技能执行效果的能力

### 技能组合（Skill Composition）

技能组合是指将多个技能按照一定逻辑顺序连接起来，共同完成复杂任务。常见的组合方式包括：

1. **线性组合**：按顺序执行一系列技能
2. **条件分支**：根据条件选择执行不同技能
3. **循环迭代**：重复执行技能直到满足条件
4. **并行执行**：同时执行多个独立的技能

## 4 工作原理

### 4.1 技能注册（Skill Registration）

技能在使用前需要向SkillManager注册，注册过程包括：

1. 创建技能实例（继承BaseSkill的具体实现）
2. 将技能实例添加到技能管理器
3. 技能管理器建立技能名称到实例的映射

注册后，Agent可以通过技能名称动态调用相应技能。

### 4.2 技能执行（Skill Execution）

技能执行流程如下：

```
1. 接收执行请求（技能名称 + 参数）
2. 验证参数合法性
3. 创建/更新执行上下文（SkillContext）
4. 调用技能的execute方法
5. 返回执行结果（SkillResult）
6. 记录执行历史
```

### 4.3 技能组合（Skill Composition）

多个技能的组合执行流程：

```
1. 定义技能链（Skill Chain）
2. 依次执行链中每个技能
3. 前一个技能的输出作为后一个技能的输入
4. 处理执行过程中的错误
5. 返回组合执行结果
```

### 4.4 上下文传递（Context Passing）

技能之间通过SkillContext共享信息：

- **agent_memory**：Agent的共享内存，可被多个技能访问
- **session_id**：当前会话标识
- **user_id**：当前用户标识
- **metadata**：元数据，用于传递额外信息

## 5 技术实现

### 5.1 架构设计

Skills系统的技术架构包含以下核心组件：

1. **BaseSkill（技能基类）**
   - 定义所有技能的统一接口
   - 包含execute、validate_params、to_dict等方法
   - 使用抽象方法模式确保子类实现核心逻辑

2. **SkillContext（执行上下文）**
   - 在技能执行过程中传递共享信息
   - 维护会话状态、用户信息等

3. **SkillResult（执行结果）**
   - 标准化技能执行结果的返回格式
   - 包含success、data、message、error等字段

4. **SkillManager（技能管理器）**
   - 负责技能的注册、注销、查询
   - 提供技能执行和组合能力
   - 维护技能执行历史

5. **SkillsEnabledAgent（技能化Agent）**
   - 整合SkillManager的Agent实现
   - 提供面向用户的技能调用接口

### 5.2 设计模式

Skills系统采用了多种设计模式：

1. **抽象工厂模式**：BaseSkill作为抽象工厂，具体技能作为产品
2. **单例模式**：SkillManager通常作为单例存在
3. **策略模式**：不同技能实现不同的执行策略
4. **模板方法模式**：BaseSkill定义执行模板，子类实现具体逻辑
5. **装饰器模式**：可通过装饰器为技能添加额外功能（如日志、缓存）

### 5.3 关键技术点

1. **参数提取**：通过Python的inspect模块自动从execute方法提取参数定义
2. **参数验证**：在执行前验证参数是否符合技能要求
3. **错误处理**：统一的异常捕获和结果封装
4. **性能监控**：记录技能执行时间和资源消耗

"""
Agent Skills系统实现
展示如何构建可扩展的技能系统
"""

# 导入类型提示和抽象基类所需的模块
from typing import Dict, List, Any, Optional, Callable, Type
from abc import ABC, abstractmethod  # ABC用于创建抽象基类，abstractmethod用于定义抽象方法
from dataclasses import dataclass, field  # dataclass简化数据类的创建，field用于配置字段
import json  # JSON序列化
import inspect  # 反射和自省，用于提取函数签名信息

# ============ 数据结构定义 ============

@dataclass
class SkillContext:
    """技能执行上下文 - 用于在技能执行过程中传递共享信息"""
    agent_memory: Dict = field(default_factory=dict)  # Agent共享内存，存储长期信息
    session_id: str = ""  # 当前会话标识，用于区分不同会话
    user_id: str = ""  # 当前用户标识
    metadata: Dict = field(default_factory=dict)  # 额外元数据

@dataclass
class SkillResult:
    """技能执行结果 - 标准化所有技能的执行返回格式"""
    success: bool  # 执行是否成功
    data: Any  # 执行返回的数据
    message: str = ""  # 执行结果的描述信息
    error: Optional[str] = None  # 错误信息（如果执行失败）
    execution_time: float = 0.0  # 执行耗时（秒）
    tokens_used: int = 0  # 使用的token数量（用于成本跟踪）

# ============ 技能基类 ============

class BaseSkill(ABC):
    """技能基类 - 所有具体技能都继承自此类"""
    
    def __init__(self):
        # 获取当前类的名称作为技能名称
        self.name = self.__class__.__name__
        # 获取类的文档字符串作为技能描述
        self.description = self.__doc__ or ""
        # 从execute方法提取参数定义
        self.parameters = self._extract_parameters()
    
    def _extract_parameters(self) -> Dict:
        """从execute方法提取参数定义"""
        # 获取execute方法的花名册（签名）
        sig = inspect.signature(self.execute)
        params = {}
        # 遍历所有参数
        for name, param in sig.parameters.items():
            if name == 'context':
                # 跳过context参数（这是所有技能都有的）
                continue
            # 构建参数信息字典
            param_info = {
                "type": "string",  # 简化处理，实际应该根据param.annotation获取
                "description": "",  # 参数描述
                "required": param.default == inspect.Parameter.empty  # 是否有默认值
            }
            params[name] = param_info
        return params
    
    @abstractmethod
    def execute(self, context: SkillContext, **kwargs) -> SkillResult:
        """执行技能 - 抽象方法，子类必须实现"""
        pass
    
    def validate_params(self, params: Dict) -> bool:
        """验证参数 - 检查必需参数是否都提供了"""
        # 找出所有必需的参数
        required_params = [
            k for k, v in self.parameters.items()
            if v.get("required", False)
        ]
        # 检查所有必需参数是否都在传入的params中
        return all(p in params for p in required_params)
    
    def to_dict(self) -> Dict:
        """转换为字典 - 用于序列化或API返回"""
        return {
            "name": self.name,
            "description": self.description,
            "parameters": self.parameters
        }

# ============ 核心技能实现 ============

class MemorySkill(BaseSkill):
    """
    记忆技能
    管理短期和长期记忆
    """
    
    def __init__(self):
        super().__init__()  # 调用父类初始化
        self.short_term_memory: List[Dict] = []  # 短期记忆（会话级）
        self.long_term_memory: Dict[str, Any] = {}  # 长期记忆（持久化）
    
    def execute(self, context: SkillContext, action: str, key: str = "", value: Any = None) -> SkillResult:
        """
        执行记忆操作
        
        参数:
            action: 操作类型 (store/recall/clear)
            key: 记忆键
            value: 记忆值
        """
        try:
            if action == "store":
                # 存储到长期记忆（需要key）或短期记忆
                if key:
                    self.long_term_memory[key] = value
                else:
                    self.short_term_memory.append(value)
                return SkillResult(success=True, data=None, message="记忆已存储")
            
            elif action == "recall":
                # 检索记忆
                if key:
                    data = self.long_term_memory.get(key)
                    return SkillResult(success=True, data=data, message="记忆已检索")
                else:
                    return SkillResult(success=True, data=self.short_term_memory, message="短期记忆已检索")
            
            elif action == "clear":
                # 清除记忆
                if key:
                    self.long_term_memory.pop(key, None)
                else:
                    self.short_term_memory.clear()
                return SkillResult(success=True, data=None, message="记忆已清除")
            
            else:
                return SkillResult(success=False, data=None, error=f"未知操作: {action}")
                
        except Exception as e:
            return SkillResult(success=False, data=None, error=str(e))

# ============ Skill管理器 ============

class SkillManager:
    """技能管理器 - 负责所有技能的生命周期管理"""
    
    def __init__(self):
        """初始化技能管理器"""
        self.skills: Dict[str, BaseSkill] = {}  # 存储所有注册技能
        self.skill_history: List[Dict] = []  # 记录技能执行历史
        self._register_builtin_skills()  # 注册内置技能
    
    def _register_builtin_skills(self):
        """注册内置技能 - 系统启动时自动注册"""
        builtin_skills = [
            MemorySkill(),
            PlanningSkill(),
            ReflectionSkill(),
            CodeGenerationSkill(),
            DataAnalysisSkill(),
            WebSearchSkill()
        ]
        
        for skill in builtin_skills:
            self.register_skill(skill)
    
    def register_skill(self, skill: BaseSkill) -> bool:
        """
        注册技能
        
        参数:
            skill: 技能实例
        """
        if skill.name in self.skills:
            print(f"⚠️ 技能'{skill.name}'已存在，将被覆盖")
        
        self.skills[skill.name] = skill
        print(f"✅ 技能已注册: {skill.name}")
        return True
    
    def unregister_skill(self, skill_name: str) -> bool:
        """注销技能 - 从管理器中移除技能"""
        if skill_name in self.skills:
            del self.skills[skill_name]
            print(f"✅ 技能已注销: {skill_name}")
            return True
        return False
    
    def get_skill(self, skill_name: str) -> Optional[BaseSkill]:
        """获取技能 - 根据名称返回技能实例"""
        return self.skills.get(skill_name)
    
    def list_skills(self) -> List[str]:
        """列出所有技能 - 返回技能名称列表"""
        return list(self.skills.keys())
    
    def execute_skill(
        self,
        skill_name: str,
        context: SkillContext,
        **params
    ) -> SkillResult:
        """
        执行技能
        
        参数:
            skill_name: 技能名称
            context: 执行上下文
            **params: 技能参数
        """
        skill = self.get_skill(skill_name)
        if not skill:
            return SkillResult(
                success=False,
                data=None,
                error=f"技能'{skill_name}'不存在"
            )
        
        # 验证参数
        if not skill.validate_params(params):
            return SkillResult(
                success=False,
                data=None,
                error="参数验证失败"
            )
        
        # 执行技能
        import time
        start_time = time.time()
        
        result = skill.execute(context, **params)
        
        execution_time = time.time() - start_time
        result.execution_time = execution_time
        
        # 记录历史
        self.skill_history.append({
            "skill": skill_name,
            "params": params,
            "result": result.success,
            "time": execution_time
        })
        
        return result
    
    def compose_skills(self, skill_chain: List[tuple], context: SkillContext) -> SkillResult:
        """
        组合执行多个技能
        
        参数:
            skill_chain: 技能链列表，格式为 [(skill_name, params), ...]
            context: 执行上下文
        """
        results = []
        
        for skill_name, params in skill_chain:
            result = self.execute_skill(skill_name, context, **params)
            results.append(result)
            
            # 如果某个技能执行失败，停止执行后续技能
            if not result.success:
                return SkillResult(
                    success=False,
                    data=results,
                    error=f"技能'{skill_name}'执行失败"
                )
        
        return SkillResult(
            success=True,
            data=results,
            message=f"成功执行{len(skill_chain)}个技能"
        )

# ============ Agent Skills集成 ============

class SkillsEnabledAgent:
    """支持技能的Agent - 整合技能系统的Agent实现"""
    
    def __init__(self):
        """初始化Agent"""
        self.skill_manager = SkillManager()  # 创建技能管理器
        self.context = SkillContext()  # 创建执行上下文
    
    def use_skill(self, skill_name: str, **params) -> SkillResult:
        """使用技能 - 代理到SkillManager的执行方法"""
        return self.skill_manager.execute_skill(skill_name, self.context, **params)
    
    def plan_and_execute(self, task: str) -> SkillResult:
        """规划并执行任务 - 展示技能组合的使用模式"""
        # 1. 规划
        plan_result = self.use_skill("PlanningSkill", task=task)
        if not plan_result.success:
            return plan_result
        
        plan = plan_result.data
        print(f"\n📋 任务规划:")
        for step in plan["steps"]:
            print(f"  - {step}")
        
        # 2. 执行（简化演示）
        # 实际应根据计划选择合适的技能执行
        
        # 3. 反思
        reflection = self.use_skill(
            "ReflectionSkill",
            action="plan_and_execute",
            result=f"任务'{task}'已规划"
        )
        
        return SkillResult(
            success=True,
            data={"plan": plan, "reflection": reflection.data},
            message="任务规划完成"

## 7 常见问题

### 7.1 技能设计问题

**Q1: 技能粒度如何把控？**
- 过于细粒度会导致类似Tool的碎片化问题
- 过于粗粒度会导致复用性降低
- 建议以"完成一个完整的功能目标"为粒度标准

**Q2: 技能之间如何避免状态污染？**
- 每个技能实例应维护独立的状态
- 使用SkillContext在技能间传递共享状态
- 定期清理不再需要的状态

**Q3: 如何处理技能的依赖关系？**
- 在SkillManager中维护技能依赖图
- 执行前进行拓扑排序确定执行顺序
- 被依赖的技能应先注册

### 7.2 性能问题

**Q4: 技能实例是否需要每次创建？？**
- 建议使用单例或对象池模式复用技能实例
- 对于有状态的技能（如MemorySkill），需考虑线程安全性

**Q5: 如何优化技能执行性能？**
- 使用异步执行处理I/O密集型技能
- 对可缓存的结果进行缓存
- 监控技能执行时间，优化瓶颈

### 7.3 扩展性问题

**Q6: 如何动态添加新技能？**
- 通过register_skill方法动态注册
- 支持从外部模块加载技能
- 可实现技能热更新

**Q7: 技能版本如何管理？**
- 在SkillResult中包含版本信息
- SkillManager支持版本查询和回滚
- 使用配置中心管理技能版本
