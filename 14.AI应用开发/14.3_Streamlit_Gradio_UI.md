# 14.3 Streamlit与Gradio UI开发

## 1 技术简介

Streamlit和Gradio是Python原生的UI框架，允许数据科学家和AI开发者仅用Python代码快速构建交互式Web应用，无需前端开发经验。Streamlit适合构建数据应用和Dashboard，Gradio更适合构建机器学习模型的演示界面。这两个框架都大大降低了AI应用UI开发的门槛，让开发者可以专注于AI能力本身。

## 2 技术背景

传统的Web应用开发需要掌握HTML、CSS、JavaScript等前端技术，以及React、Vue等前端框架，学习成本较高。对于AI开发者来说，往往不擅长前端开发，这成为AI应用快速原型验证的障碍。Streamlit和Gradio的出现解决了这一问题，它们允许开发者使用纯Python代码构建交互式Web应用，代码量大大减少，上线时间大幅缩短。

## 3 核心概念

**Streamlit组件**：Streamlit提供的UI组件，如st.text_input、st.button、st.selectbox等，用于构建用户界面。

**Gradio Interface**：Gradio的核心类，通过简单的参数配置即可创建完整的模型演示界面。

**会话状态（Session State）**：Streamlit的session_state机制，用于在用户会话期间保存和共享数据。

**缓存（Cache）**：使用@st.cache_data或@st.cache_resource装饰器缓存数据或资源，避免重复计算。

**实时交互**：两个框架都支持实时交互，用户的输入会立即触发回调函数，更新界面显示。

## 4 工作原理

Streamlit的工作原理是基于脚本重新运行模式。每次用户与界面交互（如点击按钮、输入文本），Streamlit会重新运行整个脚本，根据当前状态渲染界面。开发者使用session_state来保存跨请求的状态，使用缓存来避免重复计算。

Gradio的工作原理是基于回调函数模式。开发者定义预测函数，Gradio自动创建输入组件（如文本框、文件上传）和输出组件（如文本显示、图片展示），并将用户输入传递给预测函数，将结果渲染到输出组件。

## 6 Streamlit会话管理

### 6.1 Session State基础

Session State是Streamlit的核心特性，用于在用户会话期间保存和共享数据。

```python
import streamlit as st

st.title("Session State 使用指南")

# 1. 初始化Session State
# 检查键是否存在，不存在则初始化
if "counter" not in st.session_state:
    st.session_state.counter = 0

if "user_info" not in st.session_state:
    st.session_state.user_info = {
        "name": "",
        "logged_in": False,
        "preferences": {}
    }

# 2. 使用Session State中的数据
st.write(f"当前计数：{st.session_state.counter}")

# 3. 修改Session State
col1, col2 = st.columns(2)

with col1:
    if st.button("增加计数"):
        st.session_state.counter += 1
        st.rerun()

with col2:
    if st.button("重置计数"):
        st.session_state.counter = 0
        st.rerun()

# 4. 用户信息管理示例
st.divider()
st.subheader("用户信息管理")

name = st.text_input("请输入您的名字", key="name_input")

if st.button("保存用户信息"):
    st.session_state.user_info["name"] = name
    st.session_state.user_info["logged_in"] = True
    st.success(f"欢迎，{name}！")

if st.session_state.user_info["logged_in"]:
    st.write(f"当前用户：{st.session_state.user_info['name']}")
    if st.button("退出登录"):
        st.session_state.user_info["logged_in"] = False
        st.session_state.user_info["name"] = ""
        st.rerun()
```

### 6.2 回调函数与状态更新

使用回调函数可以更优雅地管理状态变化。

```python
import streamlit as st

st.title("回调函数示例")

# 初始化Session State
if "form_data" not in st.session_state:
    st.session_state.form_data = {
        "name": "",
        "email": "",
        "age": 0,
        "submitted": False
    }

# 定义回调函数
def update_name():
    st.session_state.form_data["name"] = st.session_state.name_input

def update_email():
    st.session_state.form_data["email"] = st.session_state.email_input

def update_age():
    st.session_state.form_data["age"] = st.session_state.age_input

def submit_form():
    st.session_state.form_data["submitted"] = True

def clear_form():
    st.session_state.form_data = {
        "name": "",
        "email": "",
        "age": 0,
        "submitted": False
    }

# 创建表单
with st.form("user_form"):
    st.write("用户注册表单")
    
    name_input = st.text_input(
        "姓名",
        value=st.session_state.form_data["name"],
        key="name_input",
        on_change=update_name
    )
    
    email_input = st.text_input(
        "邮箱",
        value=st.session_state.form_data["email"],
        key="email_input",
        on_change=update_email
    )
    
    age_input = st.number_input(
        "年龄",
        min_value=0,
        max_value=150,
        value=st.session_state.form_data["age"],
        key="age_input",
        on_change=update_age
    )
    
    submit = st.form_submit_button("提交", on_click=submit_form)
    clear = st.form_submit_button("清空", on_click=clear_form)

# 显示提交结果
if st.session_state.form_data["submitted"]:
    st.success("表单提交成功！")
    st.json(st.session_state.form_data)
```

### 6.3 组件键值管理

为组件设置唯一的key参数是避免状态混乱的关键。

```python
import streamlit as st

st.title("组件键值管理")

# 情况1：循环中创建组件需要唯一的key

cols = st.columns(3)

    with col:

        )

# 情况2：条件渲染的组件
st.subheader("示例2：条件渲染的组件")

if show_extra:
    extra_value = st.text_input("额外输入", key="extra")
    st.write(f"额外输入：{extra_value}")
else:
    st.info("勾选上方复选框显示更多选项")

# 情况3：动态添加/删除组件
st.subheader("示例3：动态组件")

if "inputs" not in st.session_state:
    st.session_state.inputs = []

# 添加新输入
if st.button("添加输入框"):
    st.session_state.inputs.append(len(st.session_state.inputs))

# 显示所有输入
for idx in st.session_state.inputs:
    col1, col2 = st.columns([4, 1])
    with col1:
        val = st.text_input(f"输入 {idx+1}", key=f"dynamic_{idx}")
    with col2:
        if st.button("删除", key=f"delete_{idx}"):
            st.session_state.inputs.remove(idx)
            st.rerun()

# 显示所有值
if st.session_state.inputs:
    st.write("所有输入：")
    for idx in st.session_state.inputs:
        st.write(f"输入 {idx+1}: {st.session_state.get(f'dynamic_{idx}', '')}")
```

## 7 缓存机制

### 7.1 数据缓存

使用@st.cache_data装饰器缓存函数返回的数据，避免重复计算。

```python
import streamlit as st
import time

st.title("缓存机制演示")

# 1. 基础缓存示例
@st.cache_data
def load_data():
    """模拟从数据库加载数据"""
    time.sleep(2)  # 模拟耗时操作
    return {"data": [1, 2, 3, 4, 5], "timestamp": time.time()}

st.header("基础缓存示例")
if st.button("加载数据"):
    with st.spinner("加载中..."):
        result = load_data()
    st.write(f"数据：{result['data']}")
    st.write(f"加载时间：{result['timestamp']}")

# 2. 带参数的缓存函数
@st.cache_data
def fetch_user_data(user_id, page=1):
    """模拟从API获取用户数据"""
    time.sleep(1)
    return {
        "user_id": user_id,
        "page": page,
        "name": f"用户{user_id}",
        "data": list(range(page * 10))
    }

st.header("带参数缓存示例")
user_id = st.number_input("用户ID", min_value=1, max_value=100, value=1)
page = st.slider("页码", 1, 10, 1)

if st.button("获取用户数据"):
    data = fetch_user_data(user_id, page)
    st.json(data)

# 3. 缓存条件失效
@st.cache_data(ttl=60)  # 60秒后缓存失效
def get_data_with_ttl():
    """带过期时间的缓存"""
    return {"time": time.time()}

st.header("缓存过期示例")
if st.button("获取数据（60秒后过期）"):
    st.write(get_data_with_ttl())

# 4. 清除缓存
@st.cache_data
def expensive_function(x):
    return x ** 2

st.header("清除缓存示例")
if st.button("计算（使用缓存）"):
    result = expensive_function(100)
    st.write(f"结果：{result}")

if st.button("清除缓存"):
    st.cache_data.clear()
    st.success("缓存已清除")
```

### 7.2 资源缓存

使用@st.cache_resource装饰器缓存模型、数据库连接等资源。

```python
import streamlit as st

st.title("资源缓存演示")

# 1. 模型缓存示例
@st.cache_resource
def load_model():
    """加载AI模型（只加载一次）"""
    # 模拟模型加载
    import numpy as np
    class MockModel:
        def predict(self, x):
            return x * 2
    return MockModel()

st.header("模型缓存示例")
model = load_model()
st.write(f"模型类型：{type(model)}")

if st.button("使用模型预测"):
    result = model.predict(42)
    st.write(f"预测结果：{result}")

# 2. 数据库连接缓存
@st.cache_resource
def get_database_connection():
    """缓存数据库连接"""
    # 模拟数据库连接
    return {"connected": True, "host": "localhost"}

st.header("数据库连接缓存示例")
if "db_connection" not in st.session_state:
```

## 11 常见问题

**问题一：页面加载缓慢**
Streamlit每次交互都会重新运行整个脚本。解决方案：使用@st.cache_data缓存不经常变化的数据，使用@st.cache_resource缓存模型等资源。

**问题二：session_state状态混乱**
session_state在某些情况下会被重置。确保正确初始化所有使用的session_state键值。

**问题三：组件ID冲突**
同一个页面上有多个相同类型的组件时，可能会出现状态混乱。解决方案：使用key参数为每个组件指定唯一标识。

**问题四：文件上传处理**
大文件上传会占用较多内存。解决方案：使用st.file_uploader时设置accept_multiple_files=False限制单次上传数量，处理完成后及时释放内存。
