# 14.2 AI聊天应用

## 1 技术简介

AI聊天应用是基于大语言模型的对话系统，支持多轮对话、上下文管理、实时流式响应。核心挑战在于会话状态管理、消息历史维护、实时通信和上下文理解。聊天应用是AI技术最直接的用户触点，需要解决交互流畅性、响应实时性、上下文连贯性等问题。WebSocket是实现实时聊天的主流技术，配合LLM的流式输出，可以提供近似实时的对话体验。

## 2 技术背景

传统的Web应用采用请求-响应模式，用户发送请求后需要等待服务器处理完成才能收到响应。对于AI聊天应用，这种模式会导致较差的用户体验，因为AI生成内容需要一定时间。WebSocket技术的出现解决了这一问题，它提供了双向通信能力，服务器可以主动向客户端推送数据。同时，LLM的流式输出能力让用户可以逐字看到AI的生成过程，提升了交互体验。

## 3 核心概念

**WebSocket**：一种在单个TCP连接上提供双向通信的协议。服务器可以主动向客户端发送消息，无需客户端轮询。

**会话管理（Session Management）**：管理用户会话状态，包括会话ID、用户信息、消息历史等。多用户场景下需要考虑会话隔离和数据安全。

**消息历史**：记录对话过程中的所有消息，用于维护上下文和生成更连贯的回复。需要合理控制历史长度以避免超出模型上下文限制。

**流式输出（Streaming）**：AI模型逐步返回生成内容，客户端逐步渲染显示。这种方式减少了首字节时间，提升用户体验。

**实时双向通信**：客户端和服务器可以随时互相发送消息，适用于需要即时反馈的场景。

## 4 工作原理

AI聊天应用的工作流程如下：客户端通过WebSocket连接到服务器；服务器为该连接创建会话，保存连接对象和会话状态；客户端发送消息，服务器接收并处理；服务器调用LLM生成响应（流式）；服务器逐步将生成的内容推送给客户端；客户端渲染显示，并保存到本地消息历史。

会活管理需要处理以下问题：会话创建和销毁、消息历史存储和清理、会话超时处理、多设备同时在线处理等。通常使用Redis等分布式存储来管理会话状态，支持多实例部署。

AI聊天应用的技术实现主要包含WebSocket服务器、消息处理、LLM调用、会话管理等方面。

class ConnectionManager:
    """WebSocket连接管理器"""
    def __init__(self):
        self.active_connections: Dict[str, WebSocket] = {}
        self.user_sessions: Dict[str, dict] = {}
    
    async def connect(self, websocket: WebSocket, client_id: str):
        """建立连接"""
        await websocket.accept()
        self.active_connections[client_id] = websocket
        self.user_sessions[client_id] = {
            "messages": [],
            "connected_at": datetime.now().isoformat()
        }
        print(f"[WebSocket] 用户 {client_id} 已连接")
    
    def disconnect(self, client_id: str):
        """断开连接"""
        if client_id in self.active_connections:
            del self.active_connections[client_id]
        if client_id in self.user_sessions:
            del self.user_sessions[client_id]
        print(f"[WebSocket] 用户 {client_id} 已断开")
    
    async def send_message(self, client_id: str, message: dict):
        """发送消息"""
        if client_id in self.active_connections:
            await self.active_connections[client_id].send_json(message)
    
    async def broadcast(self, message: dict):
        """广播消息"""
        for client_id in self.active_connections:
            await self.active_connections[client_id].send_json(message)

    client = openai.OpenAI()
    
    # 构建消息历史 - 最近5轮对话
    session = manager.user_sessions[client_id]
    messages = [
        {"role": "system", "content": "你是一个 helpful AI助手"}
    ] + [
        {"role": m["role"], "content": m["content"]}
        for m in session["messages"][-5:]
    ]
    
    # 创建流式响应
    stream = client.chat.completions.create(
        model="gpt-5.4-mini",
        messages=messages,
        stream=True,
        temperature=0.7
    )
    
    full_response = ""
    
    for chunk in stream:
        if chunk.choices[0].delta.content:
            token = chunk.choices[0].delta.content
            full_response += token
            
            # 发送token到客户端
            await manager.send_message(client_id, {
                "type": "stream",
                "token": token,
                "finished": False
            })
            
            await asyncio.sleep(0.01)  # 控制流式速度
    
    # 发送完成信号
    await manager.send_message(client_id, {
        "type": "stream",
        "token": "",
        "finished": True
    })
    
    # 保存AI回复
    session["messages"].append({
        "role": "assistant",
        "content": full_response,
        "timestamp": datetime.now().isoformat()
    })
    
    # 发送完整消息确认
    await manager.send_message(client_id, {
        "type": "complete",
        "content": full_response,
        "timestamp": datetime.now().isoformat()
    })

def complete_example():
    """完整的生产级聊天应用代码"""
    
    from typing import Dict, List, Optional
    import redis
    import json
    from datetime import datetime
    import hashlib
    
    # ========================================
    # 1. 消息存储类
    # ========================================
    class MessageStore:
        """消息存储 - 支持Redis持久化"""
        
        def __init__(self, redis_url: str = "redis://localhost:6379"):
            self.redis = redis.from_url(redis_url, decode_responses=True)
            self.ttl = 3600 * 24 * 7  # 7天过期
        
        def save_message(self, session_id: str, message: dict):
            """保存消息"""
            key = f"session:{session_id}:messages"
            self.redis.rpush(key, json.dumps(message, ensure_ascii=False))
            self.redis.expire(key, self.ttl)
        
        def get_messages(self, session_id: str, limit: int = 50) -> List[dict]:
            """获取消息历史"""
            key = f"session:{session_id}:messages"
            messages = self.redis.lrange(key, -limit, -1)
            return [json.loads(m) for m in messages]
        
        def clear_session(self, session_id: str):
            """清理会话"""
            key = f"session:{session_id}:messages"
            self.redis.delete(key)
    
    # ========================================
    # 2. 上下文管理器
    # ========================================
    class ContextManager:
        """上下文管理器 - 智能截断历史"""
        
        def __init__(self, max_tokens: int = 4000):
            self.max_tokens = max_tokens
            self.avg_token_per_char = 4  # 约4字符=1 token
        
        def build_context(
            self,
            system_prompt: str,
            history: List[dict],
            current_message: str
        ) -> List[dict]:
            """构建上下文 - 智能裁剪"""
            
            messages = [{"role": "system", "content": system_prompt}]
            
            # 估算当前消息token
            current_tokens = len(current_message) // self.avg_token_per_char
            available_tokens = self.max_tokens - current_tokens - 200  # 预留空间
            
            # 从后向前添加历史
            used_tokens = 0
            for msg in reversed(history):
                msg_tokens = len(msg.get("content", "")) // self.avg_token_per_char
                
                if used_tokens + msg_tokens > available_tokens:
                    break
                
                messages.insert(1, {
                    "role": msg["role"],
                    "content": msg["content"]
                })
                used_tokens += msg_tokens
            
            # 添加当前消息
            messages.append({"role": "user", "content": current_message})
            
            return messages

    # ========================================
    class ChatService:
        """聊天服务 - 整合所有功能"""
        
        def __init__(self):
            # 初始化组件
            self.message_store = MessageStore()
            self.context_manager = ContextManager(max_tokens=4000)
            self.openai_client = None  # 初始化OpenAI客户端
        
        def create_session(self, user_id: str) -> str:
            """创建会话"""
            session_id = hashlib.md5(
                f"{user_id}_{datetime.now()}".encode()
            ).hexdigest()[:16]
            
            # 保存会话元数据
            key = f"session:{session_id}:meta"
            self.message_store.redis.hset(key, mapping={
                "user_id": user_id,
                "created_at": datetime.now().isoformat()
            })
            self.message_store.redis.expire(key, self.message_store.ttl)
            
            return session_id
        
        async def chat(self, session_id: str, user_message: str):
            """处理聊天请求 - 返回异步生成器"""
            
            # 1. 获取历史消息
            history = self.message_store.get_messages(session_id)
            
            # 2. 构建上下文
            messages = self.context_manager.build_context(
                system_prompt="你是一个有帮助的AI助手。",
                history=history,
                current_message=user_message
            )
            
            # 3. 保存用户消息
            self.message_store.save_message(session_id, {
                "role": "user",
                "content": user_message,
                "timestamp": datetime.now().isoformat()
            })
            
            # 4. 调用LLM（流式）
            # 这里返回生成器，实际使用时yield每个token
            response_text = ""
            
            # 模拟流式输出
            for word in ["你好", "！有什么", "可以帮", "你的吗", "？"]:
                response_text += word
                yield {
                    "type": "stream",
                    "token": word,
                    "finished": False
                }
                import asyncio
                await asyncio.sleep(0.1)
            
            # 5. 保存助手消息
            self.message_store.save_message(session_id, {
                "role": "assistant",
                "content": response_text,
                "timestamp": datetime.now().isoformat()
            })
            
            # 6. 返回完成
            yield {
                "type": "complete",
                "content": response_text,
                "timestamp": datetime.now().isoformat()
            }
        
        def get_session_info(self, session_id: str) -> Optional[dict]:
            """获取会话信息"""
            key = f"session:{session_id}:meta"
            meta = self.message_store.redis.hgetall(key)
            
            if not meta:
                return None
            
            # 获取消息数量
            msg_key = f"session:{session_id}:messages"
            msg_count = self.message_store.redis.llen(msg_key)
            
            return {
                "session_id": session_id,
                "user_id": meta.get("user_id"),
                "created_at": meta.get("created_at"),
                "message_count": msg_count
            }

## 7 常见问题

**问题一：WebSocket连接中断**
网络不稳定或服务器重启会导致连接中断。解决方案：在客户端实现自动重连机制，使用指数退避策略。

**问题二：消息顺序错乱**
在并发场景下，多个消息可能乱序到达。解决方案：在消息中添加序号，客户端按序号排序后再显示。

**问题三：会话状态丢失**
服务器重启会导致内存中的会话状态丢失。解决方案：将会话状态持久化到Redis或数据库。

**问题四：流式响应中断**
网络问题可能导致流式响应中断。解决方案：实现断点续传，客户端可以从上次中断的地方继续接收。

**问题五：上下文长度限制**
模型有token限制，长对话需要截断。解决方案：使用ContextManager智能裁剪历史消息。

**问题六：多实例部署问题**
多实例部署时会话状态不同步。解决方案：使用Redis集中存储会话状态。
