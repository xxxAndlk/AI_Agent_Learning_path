# 8.5 PGVector向量扩展

## 1 技术简介

PGVector是PostgreSQL的向量扩展插件，为传统关系型数据库添加高效的向量存储和相似度搜索能力。它允许在同一个数据库中存储结构化数据和向量数据，无需额外部署专用向量数据库，降低系统复杂度。PGVector特别适合已有PostgreSQL基础设施的团队，或者需要同时处理结构化数据和向量的应用场景。

## 2 技术背景

在PGVector出现之前，如果需要向量检索能力，通常需要额外部署专门的向量数据库，如Milvus、Weaviate等。这增加了系统复杂度，需要维护多套数据库，也带来了数据同步的问题。PGVector的出现使得在PostgreSQL中直接存储和检索向量成为可能，它继承了PostgreSQL的成熟特性（如ACID事务、SQL查询、索引等），同时添加了向量能力。

## 3 核心概念

**向量类型（vector）**：PGVector定义了一种新的数据类型用于存储向量，支持任意维度的向量存储。

**距离运算符**：PGVector提供了三个距离运算符，<=>表示余弦距离，<->表示欧氏距离（也称L2距离），<#>表示内积。

**索引类型**：PGVector支持两种索引类型，ivfflat使用倒排文件索引，适合中等规模数据；hnsw使用分层导航小世界图索引，适合需要高召回率的场景。

**混合查询**：支持SQL与向量检索的组合查询，可以在WHERE子句中同时使用向量相似度过滤和普通条件过滤。

## 4 工作原理

PGVector的工作原理是在PostgreSQL的基础上添加了向量数据类型的支持和向量索引的构建。在数据存储方面，向量数据以二进制格式存储在表中，可以与普通列共存，支持完整的CRUD操作。

在索引构建方面，PGVector支持两种索引算法。IVF索引通过将向量聚类来减少搜索范围，构建时需要指定聚类数量（lists参数）；HNSW索引通过构建分层图结构来实现快速搜索，构建时需要指定连接数（m参数）和构建时的搜索宽度（ef_construction参数）。

在搜索执行方面，向量搜索可以作为普通SQL查询的一部分，支持ORDER BY向量距离、LIMIT返回数量等操作。还可以与标量条件结合，实现混合搜索。

## 5 技术实现

PGVector的实现分为安装部署和数据操作两个部分。在安装部署方面，可以使用Docker快速启动带有PGVector的PostgreSQL，或者在现有PostgreSQL上安装扩展。在Docker中运行只需要一条命令；在已有PostgreSQL上安装需要根据操作系统选择合适的安装方式，然后执行CREATE EXTENSION vector命令启用扩展。

在数据操作方面，首先需要创建包含向量列的表，可以使用DDL语句定义向量字段；然后插入向量数据，注意向量需要用数组格式表示；接着创建索引来加速搜索，可以使用HNSW或IVF索引；最后执行搜索查询，使用距离运算符计算相似度。

```python
import psycopg2
from pgvector.psycopg2 import register_vector
import numpy as np
from typing import List, Tuple, Dict, Any

class PGVectorClient:
    """PGVector客户端封装"""
    
    def __init__(self, connection_string: str = "postgresql://postgres:password@localhost:5432/postgres"):
        """
        初始化PGVector客户端
        
        参数:
            connection_string: PostgreSQL连接字符串
        """
        self.connection_string = connection_string
        self.conn = None
        self._connect()
    
    def _connect(self):
        """建立连接并注册向量类型"""
        self.conn = psycopg2.connect(self.connection_string)
        register_vector(self.conn)
        
        # 启用pgvector扩展
        with self.conn.cursor() as cur:
            cur.execute("CREATE EXTENSION IF NOT EXISTS vector;")
            self.conn.commit()
        
        print("[PGVector] 连接成功")
    
    def create_table(self, table_name: str, dimension: int = 1536):
        """
        创建向量表
        
        参数:
            table_name: 表名
            dimension: 向量维度
        """
        with self.conn.cursor() as cur:
            cur.execute(f"""
                CREATE TABLE IF NOT EXISTS {table_name} (
                    id SERIAL PRIMARY KEY,
                    content TEXT,
                    embedding vector({dimension}),
                    metadata JSONB,
                    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
                );
            """)
            
            # 创建向量索引（HNSW）
            cur.execute(f"""
                CREATE INDEX IF NOT EXISTS {table_name}_embedding_idx 
                ON {table_name} 
                USING hnsw (embedding vector_cosine_ops);
            """)
            
            self.conn.commit()
        
        print(f"[PGVector] 表 {table_name} 创建成功")
    
    def insert_vectors(
        self, 
        table_name: str, 
        contents: List[str], 
        embeddings: List[List[float]],
        metadatas: List[Dict] = None
    ):
        """
        批量插入向量
        
        参数:
            table_name: 表名
            contents: 文本内容列表
            embeddings: 向量列表
            metadatas: 元数据列表（可选）
        """
        if metadatas is None:
            metadatas = [{} for _ in contents]
        
        with self.conn.cursor() as cur:
            for content, embedding, metadata in zip(contents, embeddings, metadatas):
                cur.execute(f"""
                    INSERT INTO {table_name} (content, embedding, metadata)
                    VALUES (%s, %s, %s);
                """, (content, embedding, psycopg2.extras.Json(metadata)))
            
            self.conn.commit()
        
        print(f"[PGVector] 插入 {len(contents)} 条记录")
    
    def search_similar(
        self,
        table_name: str,
        query_embedding: List[float],
        top_k: int = 5,
        metric: str = "cosine"  # cosine, l2, inner_product
    ) -> List[Dict[str, Any]]:
        """
        相似度搜索
        
        参数:
            table_name: 表名
            query_embedding: 查询向量
            top_k: 返回数量
            metric: 距离度量方式
        
        返回:
            搜索结果列表
        """
        # 选择距离函数
        distance_func = {
            "cosine": "1 - (embedding <=> %s::vector)",  # 余弦相似度
            "l2": "embedding <-> %s::vector",           # L2距离
            "inner_product": "embedding <#> %s::vector"  # 内积
        }.get(metric, "1 - (embedding <=> %s::vector)")
        
        with self.conn.cursor() as cur:
            cur.execute(f"""
                SELECT id, content, metadata, {distance_func} as distance
                FROM {table_name}
                ORDER BY embedding <=> %s::vector
                LIMIT %s;
            """, (query_embedding, query_embedding, top_k))
            
            results = []
            for row in cur.fetchall():
                results.append({
                    "id": row[0],
                    "content": row[1],
                    "metadata": row[2],
                    "score": float(row[3]) if metric == "cosine" else float(row[3])
                })
        
        return results
    
    def hybrid_search(
        self,
        table_name: str,
        query_embedding: List[float],
        keyword: str = None,
        filter_conditions: Dict = None,
        top_k: int = 5
    ) -> List[Dict[str, Any]]:
        """
        混合搜索：向量相似度 + 关键词匹配 + 过滤条件
        
        参数:
            table_name: 表名
            query_embedding: 查询向量
            keyword: 关键词（全文搜索）
            filter_conditions: 过滤条件
            top_k: 返回数量
        """
        # 构建WHERE子句
        where_clauses = ["1=1"]
        params = [query_embedding]
        
        if keyword:
            where_clauses.append("content ILIKE %s")
            params.append(f"%{keyword}%")
        
        if filter_conditions:
            for key, value in filter_conditions.items():
                where_clauses.append(f"metadata->>{key} = %s")
                params.append(value)
        
        where_sql = " AND ".join(where_clauses)
        params.append(top_k)
        
        with self.conn.cursor() as cur:
            cur.execute(f"""
                SELECT id, content, metadata, 
                       1 - (embedding <=> %s::vector) as similarity
                FROM {table_name}
                WHERE {where_sql}
                ORDER BY embedding <=> %s::vector
                LIMIT %s;
            """, tuple(params))
            
            results = []
            for row in cur.fetchall():
                results.append({
                    "id": row[0],
                    "content": row[1],
                    "metadata": row[2],
                    "similarity": float(row[3])
                })
        
        return results
    
    def delete_by_id(self, table_name: str, doc_id: int):
        """根据ID删除"""
        with self.conn.cursor() as cur:
            cur.execute(f"DELETE FROM {table_name} WHERE id = %s;", (doc_id,))
            self.conn.commit()
    
    def close(self):
        """关闭连接"""
        if self.conn:
            self.conn.close()
            print("[PGVector] 连接已关闭")


# ============ 使用示例 ============

def pgvector_basic_example():
    """PGVector基础使用示例"""
    
    print("="*60)
    print("PGVector基础示例")
    print("="*60)
    
    # 初始化客户端
    client = PGVectorClient("postgresql://postgres:password@localhost:5432/postgres")
    
    # 创建表
    client.create_table("documents", dimension=384)
    
    # 准备示例数据
    from sentence_transformers import SentenceTransformer
    
    model = SentenceTransformer('BAAI/bge-small-zh')
    
    documents = [
        "机器学习是人工智能的一个重要分支",
        "深度学习使用神经网络进行学习",
        "Python是数据科学的主流语言",
        "PostgreSQL是强大的开源数据库",
        "向量数据库用于存储和检索高维向量"
    ]
    
    # 生成向量
    embeddings = model.encode(documents).tolist()
    
    # 元数据
    metadatas = [
        {"category": "AI", "author": "user1"},
        {"category": "AI", "author": "user2"},
        {"category": "Programming", "author": "user1"},
        {"category": "Database", "author": "user3"},
        {"category": "Database", "author": "user2"}
    ]
    
    # 插入数据
    client.insert_vectors("documents", documents, embeddings, metadatas)
    
    # 向量搜索
    query = "人工智能和机器学习"
    query_emb = model.encode(query).tolist()
    
    print(f"\n查询: {query}")
    print("-"*60)
    
    results = client.search_similar("documents", query_emb, top_k=3)
    
    print("\n向量搜索结果:")
    for i, result in enumerate(results, 1):
        print(f"{i}. [{result['score']:.4f}] {result['content']}")
    
    # 混合搜索
    print("\n" + "="*60)
    print("混合搜索（向量 + 关键词过滤）")
    print("="*60)
    
    results = client.hybrid_search(
        "documents",
        query_emb,
        keyword="学习",  # 包含"学习"关键字
        filter_conditions={"category": "AI"},  # category为AI
        top_k=3
    )
    
    print(f"\n查询: {query} (关键词: 学习, 分类: AI)")
    print("-"*60)
    for i, result in enumerate(results, 1):
        print(f"{i}. [{result['similarity']:.4f}] {result['content']}")
        print(f"   元数据: {result['metadata']}")
    
    # 关闭连接
    client.close()


if __name__ == "__main__":
    pgvector_basic_example()
```

## 7 常见问题

**问题一：安装失败**
确保PostgreSQL版本符合要求（12及以上）；根据操作系统选择正确的安装方式；安装后需要执行CREATE EXTENSION vector来启用扩展。

**问题二：索引创建慢**
HNSW索引创建时间与数据量成正比，可以调整m和ef_construction参数来平衡构建时间和索引质量；对于超大数据，考虑使用分区表。

**问题三：向量维度不匹配**
创建表时需要指定正确的维度；所有向量维度必须一致；维度定义错误会导致插入和查询失败。

**问题四：搜索结果不准确**
检查使用的距离度量是否正确（余弦距离vs欧氏距离vs内积）；确认索引参数是否合理；对于召回率要求高的场景，考虑使用HNSW索引。

## 8 应用场景

**数据库现代化**：在现有PostgreSQL基础上添加向量能力，无需引入新的数据库系统，适合已有PostgreSQL基础设施的团队。

**混合数据查询**：需要同时查询结构化数据和向量数据的场景，如同时按类别和语义相似度过滤文档。

**中小规模向量应用**：数据量在百万级以下的场景，PGVector可以提供足够的性能，同时简化系统架构。

**原型开发**：快速验证向量检索功能的可行性，验证通过后再考虑迁移到专用向量数据库。
