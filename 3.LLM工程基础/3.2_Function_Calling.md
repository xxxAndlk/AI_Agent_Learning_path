# 3.2 Function Calling

## 1 技术简介

**技术定义**：Function Calling（函数调用）是LLM的一种能力，允许模型识别需要调用外部函数或API来完成任务，并生成结构化的函数调用请求。这使得LLM能够与外部系统、数据源和工具进行交互，极大地扩展了AI应用的能力边界。

Function Calling的核心价值在于：
- **打破信息孤岛**：使LLM能够访问实时数据、内部系统和第三方服务
- **执行实际操作**：不仅生成文本，还能执行计算、查询数据库等操作
- **构建智能Agent**：是构建AI Agent的基础能力
- **增强可靠性**：通过调用确定性函数减少模型幻觉

**Function Calling的工作流程**：
1. 用户提出需要外部信息的请求
2. 模型分析请求，决定是否需要调用函数
3. 模型生成函数调用请求（包含函数名和参数）
4. 系统执行函数并返回结果
5. 模型基于函数结果生成最终回答

## 2 技术背景

**为什么需要Function Calling**：

1. **模型知识的局限性**：
   - LLM的训练数据有截止日期，无法获取实时信息
   - 私有数据和内部系统信息对模型不可见
   - 模型无法执行计算或访问外部服务

2. **构建智能应用的需求**：
   - 简单的问答无法满足实际业务需求
   - 需要与数据库、API、业务系统深度集成
   - 需要执行多步骤的复杂任务

3. **技术演进**：
   - 2023年OpenAI在GPT-4中首次引入Function Calling
   - 2024年Anthropic、Google、国产厂商（DeepSeek、Qwen）全面支持此能力
   - 2025年OpenAI推出 **Responses API**，将 Function Calling、结构化输出、网络搜索、文件检索整合为统一接口，成为官方推荐的新范式
   - 成为构建AI Agent的核心技术

> **2025年新趋势**：OpenAI 已官方推荐新项目使用 **Responses API**（`client.responses.create`）替代传统的 Chat Completions + Assistants API。Responses API 内置了 `web_search_preview`、`file_search` 等官方工具，并原生支持推理模型（`o3`、`o4-mini`）的函数调用。

**Function Calling vs Plugin vs Responses API**：
- Function Calling：模型生成结构化调用请求，由应用层执行
- Plugin：预定义的函数直接暴露给模型，模型选择调用哪个
- Responses API：OpenAI 2025 年推出的统一接口，内置官方工具、原生支持推理模型和结构化输出

## 3 核心概念

**Tool（工具）**：
在Function Calling中，工具是预先定义的函数，模型可以请求调用它们。每个工具包含：
- name：函数名称
- description：函数功能描述
- parameters：参数定义（JSON Schema格式）

**Responses API 中的官方工具（内置）**：
从2025年开始，OpenAI Responses API 提供了无需开发者自己实现的官方工具：
- `web_search_preview`：实时网络搜索
- `file_search`：基于向量库的文件检索
- `computer_use_preview`：模拟用户操作电脑

**推理模型的Function Calling**：
`o3`、`o4-mini`、`DeepSeek-R1`、`Claude 4`等推理模型在2024-2025年逐渐支持Function Calling，其特点是：
- 先在“思维链”中规划调用逻辑，再生成结构化调用请求
- 适合复杂多步骤的Agent任务，但延迟更高（通常数秐至数十秒）
- 需要通过 `reasoning_effort` 或类似参数控制推理强度

**JSON Schema**：
用于定义函数参数的结构化格式，包含：
- type：参数类型（string、integer、boolean等）
- description：参数说明
- enum：枚举值列表
- required：必填参数列表

**Tool Choice**：
控制模型如何选择调用工具：
- auto：模型自动决定是否调用
- none：禁止调用工具
- 强制调用：指定必须调用某个特定函数

**Function Call流程的关键阶段**：
1. **工具定义阶段**：声明可用的函数及其参数
2. **推理阶段**：模型决定是否调用及调用哪个函数
3. **执行阶段**：应用层执行函数调用
4. **响应阶段**：将函数结果返回给模型生成最终回答

## 4 工作原理

**Function Calling的完整流程**：

```
用户请求 → 第一次API调用（带工具定义）
              ↓
         模型分析请求
              ↓
    ┌────────┴────────┐
    ↓                 ↓
 不需要调用函数    需要调用函数
    ↓                 ↓
 直接生成回答    返回函数调用请求
                     ↓
              应用执行函数
                     ↓
              第二次API调用（带函数结果）
                     ↓
              生成最终回答
```

**模型决策逻辑**：
1. 分析用户请求是否需要外部信息
2. 评估可用的工具是否能满足需求
3. 选择最合适的工具并提取参数
4. 生成符合参数格式的调用请求

**工具定义的最佳实践**：
- description要清晰描述函数功能
- 参数说明要具体，便于模型理解
- 使用enum限制可选值范围
- 只暴露必要的参数，减少复杂性

## 5 技术实现

**定义工具的基本结构**：
```json
{
  "type": "function",
  "function": {
    "name": "函数名",
    "description": "函数功能描述",
    "parameters": {
      "type": "object",
      "properties": {...},
      "required": [...]
    }
  }
}
```

**Function Calling的实现要点**：
1. 合理设计工具和参数
2. 处理模型可能不调用函数的情况
3. 正确解析函数调用请求
4. 异步处理可能耗时的函数调用
5. 错误处理和降级策略

## 6 Python代码实现与逐行解析

```python
import json                        # JSON处理库
import os                          # 操作系统接口
from openai import OpenAI          # OpenAI官方客户端

# 从环境变量读取API密钥初始化客户端（安全做法，避免硬编码）
client = OpenAI(api_key=os.getenv("OPENAI_API_KEY"))

def get_current_weather(location, unit="摄氏度"):
    """模拟天气查询函数
    
    参数:
        location: 城市名称
        unit: 温度单位（摄氏度/华氏度）
    返回:
        JSON格式的天气信息
    """
    # 模拟天气数据库
    weather_data = {
        "北京": {"温度": 22, "单位": unit, "描述": "晴朗"},
        "上海": {"温度": 25, "单位": unit, "描述": "多云"},
        "深圳": {"温度": 28, "单位": unit, "描述": "小雨"}
    }
    # 查询并返回JSON格式数据
    return json.dumps(weather_data.get(location, {"温度": "未知", "描述": "未知"}))

def function_calling_example():
    """Function Calling完整流程示例
    
    让模型能够调用外部函数获取信息
    """
    # 定义可用的工具（函数）
    tools = [
        {
            "type": "function",
            "function": {
                "name": "get_current_weather",      # 函数名
                "description": "获取指定城市的当前天气",  # 函数描述
                "parameters": {                      # 参数定义（JSON Schema格式）
                    "type": "object",
                    "properties": {
                        "location": {                # 位置参数
                            "type": "string",
                            "description": "城市名称，如北京、上海",
                        },
                        "unit": {                    # 单位参数
                            "type": "string",
                            "enum": ["摄氏度", "华氏度"],  # 枚举值
                        },
                    },
                    "required": ["location"],        # 必填参数
                },
            },
        }
    ]
    
    # 用户询问天气
    messages = [{"role": "user", "content": "今天北京的天气怎么样？"}]
    
    # 第一次调用：让模型决定是否需要调用函数
    response = client.chat.completions.create(
        model="gpt-4o-mini",
        messages=messages,
        tools=tools,                       # 提供可用工具
        tool_choice="auto",                # 自动决定是否调用工具
    )
    
    response_message = response.choices[0].message
    tool_calls = response_message.tool_calls  # 模型请求的函数调用
    
    # 检查模型是否决定调用函数
    if tool_calls:
        # 可用的函数映射表
        available_functions = {
            "get_current_weather": get_current_weather,
        }
        
        # 将模型的响应添加到消息历史
        messages.append(response_message)
        
        # 执行每个函数调用
        for tool_call in tool_calls:
            function_name = tool_call.function.name        # 获取函数名
            function_to_call = available_functions[function_name]  # 获取函数
            function_args = json.loads(tool_call.function.arguments)  # 解析参数
            
            # 调用函数并获取结果
            function_response = function_to_call(
                location=function_args.get("location"),
                unit=function_args.get("unit"),
            )
            
            # 将函数结果添加到消息历史
            messages.append(
                {
                    "tool_call_id": tool_call.id,         # 工具调用ID
                    "role": "tool",                        # 角色标记为工具
                    "name": function_name,                 # 函数名
                    "content": function_response,          # 函数返回结果
                }
            )
        
        # 第二次调用：让模型基于函数结果生成最终回答
        second_response = client.chat.completions.create(
            model="gpt-4o-mini",
            messages=messages,
        )
        print("Function Calling结果:", second_response.choices[0].message.content)

if __name__ == "__main__":
    # 需要先初始化client
    # function_calling_example()
    print("请先初始化OpenAI客户端后再运行")
```

## 7 常见问题

**Function Calling常见问题**：

1. **模型不调用函数**
   - 原因：提示不够清晰或工具定义不明确
   - 解决：优化工具description，明确说明何时使用

2. **参数提取错误**
   - 原因：参数定义不够具体
   - 解决：详细描述每个参数的含义和格式

3. **函数调用失败**
   - 原因：函数执行出错或超时
   - 解决：添加错误处理，返回有意义的错误信息

4. **多次调用不生效**
   - 原因：消息历史处理不当
   - 解决：正确维护messages列表，包含所有交互

5. **工具选择不当**
   - 原因：多个工具功能重叠
   - 解决：明确区分工具职责，避免功能冲突

   - 查询商品库存

   - 计算运费

   - 访问ERP数据
