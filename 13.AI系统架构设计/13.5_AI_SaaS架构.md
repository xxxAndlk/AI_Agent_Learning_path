# 13.5 AI SaaS架构

## 1 技术简介

AI SaaS架构是指为多个租户提供AI服务的软件架构模式。在AI SaaS应用中，需要同时处理多个用户的请求，同时保证数据隔离和资源公平分配。AI SaaS架构需要考虑多租户隔离、资源管理、计费计量、安全合规等企业级特性，是构建商业化AI产品的基础。

## 2 技术背景

随着AI技术的普及，越来越多的企业希望使用AI能力但缺乏自建能力，这催生了AI SaaS（Software as a Service）服务的需求。AI SaaS允许用户通过订阅或按量付费的方式使用AI服务，无需自行搭建基础设施。然而，为多个租户提供AI服务面临诸多挑战，包括：如何保证租户间的数据隔离、如何公平分配计算资源、如何实现灵活的计费策略、如何满足数据安全合规要求等。这些挑战推动了AI SaaS架构的发展。

## 3 核心概念

**多租户（Multi-Tenant）**：一种软件架构模式，多个租户共享同一套系统资源，但相互之间数据隔离。租户可以是企业、部门或个人用户。

**租户隔离策略**：保证租户间数据和应用隔离的策略，常见的有独立数据库、独立Schema、行级隔离三种方式。

**资源配额（Quota）**：对每个租户可用资源的限制，如API调用次数、存储空间、并发数等。

**计费模型**：AI SaaS的收费模式，常见的有订阅制、按量付费、混合模式。

**API密钥（API Key）**：租户访问服务的凭证，用于身份认证和使用量统计。

**Token消耗**：LLM调用的计费单位，通常根据输入和输出的token数量计费。

## 4 工作原理

AI SaaS架构的工作流程如下：用户通过订阅或注册获得API密钥；用户使用API密钥调用AI服务；系统验证API密钥的有效性和配额；请求进入服务队列等待处理；服务处理请求并记录使用量；返回响应给用户并更新配额。

多租户隔离的实现方式有三种：独立数据库方案为每个租户创建独立的数据库实例，数据隔离最彻底但成本最高；独立Schema方案在共享数据库中为每个租户创建独立Schema，隔离性较好且成本适中；行级隔离方案在共享数据库中通过租户ID字段区分数据，实现最简单但隔离性较弱。

## 5 技术实现

AI SaaS架构的技术实现主要包含以下几个方面。API网关负责身份验证、请求路由、限流和日志记录；服务层处理具体的AI业务逻辑，如LLM调用、RAG检索等；数据层管理租户数据和向量存储；计量系统记录每个租户的资源使用情况；计费系统根据使用量计算费用。

## 6 Python代码实现与逐行解析

```python
"""
AI SaaS架构

多租户策略：
- 数据库隔离：独立数据库，成本最高，安全性最高
- Schema隔离：独立Schema，成本中等
- 租户ID：行级隔离，成本最低
"""

from typing import Dict, List, Optional
from datetime import datetime
import time


class TenantManager:
    """租户管理器"""
    
    def __init__(self):
        # 租户信息存储
        self.tenants: Dict[str, Dict] = {}
        
        # API密钥到租户ID的映射
        self.api_key_to_tenant: Dict[str, str] = {}
        
        # 配额使用记录
        self.usage_records: Dict[str, Dict] = {}
    
    def create_tenant(
        self,
        name: str,
        plan: str = "free",
        quota: Dict = None
    ) -> str:
        """创建租户"""
        tenant_id = f"tenant_{int(time.time())}"
        
        # 默认配额
        default_quota = {
            "free": {"requests_per_day": 100, "storage_mb": 100},
            "pro": {"requests_per_day": 10000, "storage_mb": 10000},
            "enterprise": {"requests_per_day": -1, "storage_mb": -1}
        }
        
        self.tenants[tenant_id] = {
            "id": tenant_id,
            "name": name,
            "plan": plan,
            "quota": quota or default_quota.get(plan, {}),
            "created_at": datetime.now().isoformat()
        }
        
        # 生成API密钥
        api_key = f"sk_{tenant_id}_{int(time.time())}"
        self.api_key_to_tenant[api_key] = tenant_id
        
        # 初始化使用记录
        self.usage_records[tenant_id] = {
            "requests_today": 0,
            "last_reset": datetime.now().date(),
            "total_requests": 0,
            "total_tokens": 0
        }
        
        return tenant_id
    
    def get_tenant_by_api_key(self, api_key: str) -> Optional[Dict]:
        """通过API密钥获取租户"""
        tenant_id = self.api_key_to_tenant.get(api_key)
        if tenant_id:
            return self.tenants.get(tenant_id)
        return None
    
    def check_quota(self, tenant_id: str, requested: int = 1) -> bool:
        """检查配额"""
        tenant = self.tenants.get(tenant_id)
        if not tenant:
            return False
        
        usage = self.usage_records.get(tenant_id, {})
        
        # 检查每日限制
        quota = tenant.get("quota", {})
        daily_limit = quota.get("requests_per_day", -1)
        
        if daily_limit > 0:
            if usage.get("requests_today", 0) + requested > daily_limit:
                return False
        
        return True
    
    def record_usage(self, tenant_id: str, tokens: int = 0):
        """记录使用量"""
        if tenant_id not in self.usage_records:
            self.usage_records[tenant_id] = {
                "requests_today": 0,
                "last_reset": datetime.now().date(),
                "total_requests": 0,
                "total_tokens": 0
            }
        
        usage = self.usage_records[tenant_id]
        
        # 检查是否需要重置每日计数
        today = datetime.now().date()
        if usage["last_reset"] != today:
            usage["requests_today"] = 0
            usage["last_reset"] = today
        
        usage["requests_today"] += 1
        usage["total_requests"] += 1
        usage["total_tokens"] += tokens
    
    def get_usage_summary(self, tenant_id: str) -> Dict:
        """获取使用摘要"""
        return self.usage_records.get(tenant_id, {})


class AISaaSService:
    """AI SaaS服务"""
    
    def __init__(self):
        self.tenant_manager = TenantManager()
    
    def verify_request(self, api_key: str) -> Optional[Dict]:
        """验证请求"""
        tenant = self.tenant_manager.get_tenant_by_api_key(api_key)
        if not tenant:
            raise ValueError("无效的API密钥")
        
        if not self.tenant_manager.check_quota(tenant["id"]):
            raise ValueError("超出配额限制")
        
        return tenant
    
    def process_request(
        self,
        api_key: str,
        prompt: str,
        model: str = "gpt-4o-mini",
        **kwargs
    ) -> Dict:
        """处理AI请求"""
        # 验证请求
        tenant = self.verify_request(api_key)
        
        # 模拟LLM调用
        response = f"处理请求: {prompt[:50]}..."
        
        # 估算token消耗
        tokens_used = len(prompt) // 4 + len(response) // 4
        
        # 记录使用量
        self.tenant_manager.record_usage(tenant["id"], tokens_used)
        
        return {
            "response": response,
            "model": model,
            "tokens_used": tokens_used,
            "tenant_id": tenant["id"]
        }


def main():
    """主函数"""
    # 创建SaaS服务
    service = AISaaSService()
    
    # 创建租户
    tenant_id = service.tenant_manager.create_tenant(
        name="示例公司",
        plan="pro"
    )
    
    print(f"创建租户: {tenant_id}")
    
    # 获取API密钥（实际应返回给租户）
    api_key = None
    for key, tid in service.tenant_manager.api_key_to_tenant.items():
        if tid == tenant_id:
            api_key = key
            break
    
    print(f"API密钥: {api_key}")
    
    # 处理请求
    response = service.process_request(
        api_key=api_key,
        prompt="帮我写一首诗"
    )
    
    print(f"响应: {response}")
    
    # 查看使用量
    usage = service.tenant_manager.get_usage_summary(tenant_id)
    print(f"使用量: {usage}")


if __name__ == "__main__":
    main()
```

def multi_tenant_design():
    """多租户设计详解"""
    print("""
    多租户设计是AI SaaS架构的核心。需要考虑如何隔离不同租户的数据和资源，
    同时保持系统的效率和成本效益。
    
    多租户隔离级别：
    
    1. 独立数据库（完全隔离）
       - 每个租户有独立的数据库实例
       - 最高安全性，完全隔离
       - 成本最高，管理复杂
       - 适合：对数据安全要求极高的企业客户
    
    2. 独立Schema（逻辑隔离）
       - 共享数据库，不同租户使用不同Schema
       - 较好的隔离性，成本适中
       - 管理相对简单
       - 适合：中等规模的企业客户
    
    3. 行级隔离（共享数据库）
       - 共享数据库，通过tenant_id区分
       - 成本最低，实现简单
       - 隔离性较弱，需要应用层保证
       - 适合：小型客户或个人用户
    
    4. 命名空间隔离
       - 租户使用独立的键空间（如Redis namespace）
       - 适合缓存层、向量存储等
    
    数据隔离的实现：
    
    1. 应用层隔离
       class TenantContext:
           def __init__(self, tenant_id: str):
               self.tenant_id = tenant_id
           
           def get_tenant_id(self) -> str:
               return self.tenant_id
    
    2. 查询层面
       def get_tenant_data(tenant_id):
           return db.query("SELECT * FROM data WHERE tenant_id = ?", tenant_id)
    
    3. 存储层面
       # 每个租户独立的存储路径
       storage_path = /data/{tenant_id}/
    
    选择隔离级别考虑因素：
    
    1. 安全合规要求
       - 金融、医疗行业需要更高级别隔离
       - 需要符合GDPR等行业法规
    
    2. 成本预算
       - 独立数据库成本最高
       - 需要平衡隔离性和成本
    
    3. 租户规模
       - 大客户可使用独立实例
       - 小客户共享资源
    
    4. 技术复杂度
       - 隔离级别越高，实现越复杂
       - 需要更多运维工作
    """)

def permission_management():
    """权限管理系统详解"""
    print("""
    权限管理是AI SaaS系统的重要组成部分，需要细粒度地控制
    不同用户对系统资源的访问权限。
    
    权限模型（RBAC）：
    
    1. 角色（Role）
       - 管理员（Admin）：完全权限
       - 开发者（Developer）：调用API权限
       - 查看者（Viewer）：只读权限
       - 自定义角色
    
    2. 权限（Permission）
       - API调用权限
       - 数据读写权限
       - 管理功能权限
       - 计费查看权限
    
    3. 用户-角色关联
       - 一个用户可以有多个角色
       - 权限可以继承
    
    权限层级：
    
    1. 组织级别
       - 公司/部门级别的权限
       - 可以管理子用户
    
    2. 应用级别
       - 同一组织下的不同应用
       - 可以设置不同的访问策略
    
    3. API级别
       - 具体的API调用权限
       - 可以限制特定模型调用
    
    实现示例：
    
    class Permission:
        def __init__(self, resource: str, action: str):
            self.resource = resource
            self.action = action  # read, write, delete
    
    class Role:
        def __init__(self, name: str):
            self.name = name
            self.permissions: List[Permission] = []
        
        def add_permission(self, permission: Permission):
            self.permissions.append(permission)
        
        def has_permission(self, resource: str, action: str) -> bool:
            return any(
                p.resource == resource and p.action == action
                for p in self.permissions
            )
    
    class User:
        def __init__(self, user_id: str):
            self.user_id = user_id
            self.roles: List[Role] = []
        
        def assign_role(self, role: Role):
            self.roles.append(role)
        
        def can_access(self, resource: str, action: str) -> bool:
            return any(role.has_permission(resource, action) for role in self.roles)
    
    API密钥权限：
    
    1. 密钥级别权限
       - 可以绑定到特定角色
       - 可以限制调用IP
       - 可以设置过期时间
    
    2. 密钥用途分类
       - 开发环境密钥（限制多）
       - 生产环境密钥（权限高）
       - 测试密钥（有配额限制）
    
    审计日志：
    
    1. 记录内容
       - 谁在什么时间访问了什么资源
       - 执行了什么操作
       - 操作结果
    
    2. 日志保留
       - 合规要求保留期限
       - 安全事件调查需要
    """)

def billing_system():
    """计费系统设计详解"""
    print("""
    计费系统是AI SaaS商业化的核心，需要准确计量资源使用并计算费用。
    
    计费模型：
    
    1. 订阅制（Subscription）
       - 按月/年付费
       - 固定费用，限定用量
       - 超出部分单独计费
       - 适合：稳定需求的企业
    
    2. 按量付费（Pay-as-you-go）
       - 实际使用量计费
       - 灵活，无固定成本
       - 适合：需求波动的场景
    
    3. 混合模式
       - 基础订阅 + 超额按量
       - 兼顾稳定性和灵活性
       - 目前最常见的模式
    
    4. 套餐模式
       - 预定义多个套餐
       - 不同套餐不同权益
       - 简化选择过程
    
    计量维度：
    
    1. Token消耗
       - 输入Token价格
       - 输出Token价格（通常更贵）
       - 不同模型不同价格
    
    2. API调用次数
       - 按请求次数计费
       - 适合简单按次收费
    
    3. 存储空间
       - 按GB/天计费
       - 向量数据存储费用
    
    4. 计算资源
       - GPU使用时间
       - 推理计算量
    
    计费实现：
    
    class BillingService:
        def __init__(self):
            self.rates = {
                "gpt-4o-mini": {"input": 0.001, "output": 0.002},  # 每1K tokens
                "gpt-4": {"input": 0.03, "output": 0.06}
            }
        
        def calculate_cost(
            self,
            model: str,
            input_tokens: int,
            output_tokens: int
        ) -> float:
            rate = self.rates.get(model, {"input": 0, "output": 0})
            return (input_tokens / 1000) * rate["input"] + \\
                   (output_tokens / 1000) * rate["output"]
    
    价格策略示例：
    
    | 套餐    | 月费   | Token配额 | 超出价格    |
    |---------|--------|-----------|-------------|
    | 免费    | $0     | 100K      | $0.03/1K    |
    | 基础    | $29    | 500K      | $0.02/1K    |
    | 专业    | $99    | 2M        | $0.015/1K   |
    | 企业    | 自定义  | 自定义     | 定制价格    |
    
    账单功能：
    
    1. 实时使用量
       - 控制台实时显示使用量
       - 预算预警
    
    2. 账单生成
       - 按月生成账单
       - 详细使用明细
    
    3. 发票管理
       - 电子发票
       - 发票开具
    
    4. 欠费处理
       - 余额预警
       - 服务限制
       - 停止服务
    """)

def rate_limiting():
    """API限流设计详解"""
    print("""
    API限流是保护系统稳定性的重要手段，防止单个租户过度使用资源
    影响其他租户的服务质量。
    
    限流策略：
    
    1. 固定窗口
       - 固定时间窗口内限制请求数
       - 实现简单，可能有临界问题
       - 适合：简单限流场景
    
    2. 滑动窗口
       - 实时滑动的窗口
       - 更平滑的限流效果
       - 适合：需要平滑限制的场景
    
    3. 令牌桶
       - 以固定速率添加令牌
       - 突发流量可以消耗令牌
       - 适合：允许突发流量的场景
    
    4. 漏桶
       - 以固定速率处理请求
       - 超出部分排队或拒绝
       - 适合：严格流量控制的场景
    
    限流维度：
    
    1. 按租户限流
       - 每个租户独立的限流配置
       - 根据套餐级别不同
    
    2. 按API限流
       - 不同的API不同的限制
       - 敏感API更严格
    
    3. 按用户限流
       - 同一租户下的不同用户
       - 可以设置用户级别限制
    
    实现示例（令牌桶）：
    
    import time
    from threading import Lock
    
    class TokenBucket:
        def __init__(self, capacity: int, refill_rate: float):
            self.capacity = capacity
            self.tokens = capacity
            self.refill_rate = refill_rate
            self.last_refill = time.time()
            self.lock = Lock()
        
        def consume(self, tokens: int = 1) -> bool:
            with self.lock:
                self._refill()
                if self.tokens >= tokens:
                    self.tokens -= tokens
                    return True
                return False
        
        def _refill(self):
            now = time.time()
            elapsed = now - self.last_refill
            tokens_to_add = elapsed * self.refill_rate
            self.tokens = min(self.capacity, self.tokens + tokens_to_add)
            self.last_refill = now
    
    多层限流：
    
    1. 接入层限流
       - API网关级别
       - 首次拦截
    
    2. 应用层限流
       - 服务级别
       - 业务逻辑限流
    
    3. 资源限流
       - 数据库连接池
       - GPU资源
    
    限流响应：
    
    1. 返回429状态码
       - Too Many Requests
       - 包含Retry-After头
    
    2. 返回配额信息
       - X-RateLimit-Limit
       - X-RateLimit-Remaining
       - X-RateLimit-Reset
    
    3. 友好提示
       - 告知用户配额情况
       - 建议升级套餐
    """)

def data_isolation():
    """数据隔离与安全详解"""
    print("""
    数据安全是AI SaaS的生命线，需要从多个层面保障租户数据的安全。
    
    数据隔离：
    
    1. 存储隔离
       - 不同租户数据物理/逻辑分离
       - 独立存储路径
       - 加密存储
    
    2. 传输隔离
       - 使用HTTPS加密传输
       - VPN或专用线路（企业版）
    
    3. 内存隔离
       - 租户数据不混存在内存
       - 及时清理
    
    安全措施：
    
    1. 身份认证
       - API密钥认证
       - OAuth 2.0
       - JWT Token
       - 多因素认证（MFA）
    
    2. 访问控制
       - 最小权限原则
       - 权限验证
       - 操作审计
    
    3. 数据加密
       - 静态数据加密（AES-256）
       - 传输加密（TLS 1.3）
       - 密钥管理（KMS）
    
    4. 安全审计
       - 操作日志
       - 异常检测
       - 入侵检测
    
    数据脱敏：
    
    1. 日志脱敏
       - 脱敏敏感信息
       - 只保留必要信息
    
    2. 错误信息脱敏

def enterprise_saas_platform():
    """完整的企业级AI SaaS平台示例"""
    from typing import Dict, List, Optional
    from datetime import datetime, timedelta
    import hashlib
    
    # ========================================
    # 1. 完整的租户模型
    # ========================================
    class Tenant:
        """租户模型"""
        
        def __init__(
            self,
            id: str,
            name: str,
            plan: str,
            quota: Dict,
            status: str = "active"
        ):
            self.id = id
            self.name = name
            self.plan = plan
            self.quota = quota  # 配额
            self.status = status
            self.created_at = datetime.now()
    
    # ========================================
    # 2. API密钥管理
    # ========================================
    class APIKeyManager:
        """API密钥管理"""
        
        def __init__(self):
            self.keys: Dict[str, Dict] = {}
        
        def generate_key(self, tenant_id: str, name: str = "默认密钥") -> str:
            """生成API密钥"""
            key = f"sk_{hashlib.md5(f'{tenant_id}_{datetime.now()}'.encode()).hexdigest()}"
            self.keys[key] = {
                "tenant_id": tenant_id,
                "name": name,
                "created_at": datetime.now(),
                "last_used": None,
                "is_active": True
            }
            return key
        
        def validate_key(self, key: str) -> Optional[Dict]:
            """验证密钥"""
            if key not in self.keys:
                return None
            key_info = self.keys[key]
            if not key_info["is_active"]:
                return None
            return key_info
        
        def revoke_key(self, key: str):
            """撤销密钥"""
            if key in self.keys:
                self.keys[key]["is_active"] = False
    
    # ========================================
    # 3. 配额管理器
    # ========================================
    class QuotaManager:
        """配额管理器"""
        
        PLANS = {
            "free": {
                "daily_requests": 100,
                "monthly_tokens": 100000,
                "storage_mb": 100,
                "concurrent_requests": 1
            },
            "pro": {
                "daily_requests": 10000,
                "monthly_tokens": 2000000,
                "storage_mb": 10000,
                "concurrent_requests": 10
            },
            "enterprise": {
                "daily_requests": -1,  # 无限制
                "monthly_tokens": -1,
                "storage_mb": -1,
                "concurrent_requests": -1
            }
        }
        
        def check_quota(self, tenant: Tenant, request_tokens: int) -> bool:
            """检查配额"""
            quota = tenant.quota
            # 这里简化，实际需要更复杂的配额检查逻辑
            return True
    
    # ========================================
    # 4. 使用量追踪
    # ========================================
    class UsageTracker:
        """使用量追踪"""
        
        def __init__(self):
            self.daily_usage: Dict[str, Dict] = {}
            self.monthly_usage: Dict[str, Dict] = {}
        
        def record_usage(
            self,
            tenant_id: str,
            request_tokens: int,
            response_tokens: int
        ):
            """记录使用量"""
            today = datetime.now().date()
            month = datetime.now().strftime("%Y-%m")
            
            # 初始化
            if tenant_id not in self.daily_usage:
                self.daily_usage[tenant_id] = {}
            if tenant_id not in self.monthly_usage:
                self.monthly_usage[tenant_id] = {}
            
            # 记录每日
            if today not in self.daily_usage[tenant_id]:
                self.daily_usage[tenant_id][today] = {
                    "requests": 0,
                    "input_tokens": 0,
                    "output_tokens": 0
                }
            self.daily_usage[tenant_id][today]["requests"] += 1
            self.daily_usage[tenant_id][today]["input_tokens"] += request_tokens
            self.daily_usage[tenant_id][today]["output_tokens"] += response_tokens
            
            # 记录每月
            if month not in self.monthly_usage[tenant_id]:
                self.monthly_usage[tenant_id][month] = {
                    "requests": 0,
                    "input_tokens": 0,
                    "output_tokens": 0
                }
            self.monthly_usage[tenant_id][month]["requests"] += 1
            self.monthly_usage[tenant_id][month]["input_tokens"] += request_tokens
            self.monthly_usage[tenant_id][month]["output_tokens"] += response_tokens
    
    # ========================================
    # 5. 完整的AI SaaS服务
    # ========================================
    class EnterpriseAISaaS:
        """企业级AI SaaS服务"""
        
        def __init__(self):
            self.tenants: Dict[str, Tenant] = {}
            self.api_key_manager = APIKeyManager()
            self.quota_manager = QuotaManager()
            self.usage_tracker = UsageTracker()
        
        def create_tenant(
            self,
            name: str,
            plan: str = "free"
        ) -> tuple[str, str]:
            """创建租户并生成API密钥"""
            import uuid
            tenant_id = str(uuid.uuid4())
            quota = self.quota_manager.PLANS.get(plan, self.quota_manager.PLANS["free"])
            
            tenant = Tenant(tenant_id, name, plan, quota)
            self.tenants[tenant_id] = tenant
            
            api_key = self.api_key_manager.generate_key(tenant_id)
            
            return tenant_id, api_key
        
        def process_request(
            self,
            api_key: str,
            prompt: str,
            model: str = "gpt-4o-mini"
        ) -> Dict:
            """处理AI请求"""
            # 1. 验证API密钥
            key_info = self.api_key_manager.validate_key(api_key)
            if not key_info:
                raise ValueError("无效的API密钥")
            
            tenant_id = key_info["tenant_id"]
            tenant = self.tenants.get(tenant_id)
            if not tenant or tenant.status != "active":
                raise ValueError("租户不存在或已停用")
            
            # 2. 检查配额
            input_tokens = len(prompt) // 4
            if not self.quota_manager.check_quota(tenant, input_tokens):
                raise ValueError("超出配额限制")
            
            # 3. 处理请求（这里简化为模拟）
            output_tokens = 100
            response = f"模拟响应：{prompt[:50]}..."
            
            # 4. 记录使用量
            self.usage_tracker.record_usage(
                tenant_id, input_tokens, output_tokens
            )
            
            return {
                "response": response,
                "model": model,
                "usage": {
                    "input_tokens": input_tokens,
                    "output_tokens": output_tokens
                }
            }
        
        def get_usage_report(self, tenant_id: str, period: str = "daily") -> Dict:
            """获取使用报告"""
            if period == "daily":

            else:
                return self.monthly_usage.get(tenant_id, {})
    
    # ========================================
    # 6. 使用示例
    # ========================================
    def main():

        # 创建租户

        print(f"API密钥: {api_key}")

        # 查看使用报告
        report = saas.get_usage_report(tenant_id)
        print(f"\n使用报告: {report}")
    
    print("企业级AI SaaS平台示例")
    print("需要配置LLM后运行完整功能")


if __name__ == "__main__":

## 7 常见问题

**问题一：数据隔离不完善**
确保选择合适的隔离策略并正确实现。数据库隔离最安全但成本高，行级隔离最经济但需要确保应用层不会发生数据泄露。

**问题二：配额控制不准确**
需要在多个层面进行配额控制：API网关层、应用层、数据库层。确保配额检查的原子性，避免并发情况下的超额使用。

**问题三：计费不准确**
详细记录每个请求的使用量，包括token消耗、API调用次数、存储使用等。提供使用明细报表，让租户能够核对。

**问题四：安全性**
API密钥需要安全存储和传输；敏感数据需要加密；需要防范SQL注入等常见安全威胁。

**问题五：扩展性不足**
系统设计时需要考虑未来的扩展需求，包括：数据库分片、缓存策略、服务无状态化等。

**问题六：多租户性能**
共享资源时，一个租户的高负载可能影响其他租户。需要实现资源隔离和公平调度。
