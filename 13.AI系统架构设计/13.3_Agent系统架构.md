# 13.3 Agent系统架构

## 1 技术简介

Agent系统架构是指能够自主理解任务目标、规划执行步骤、调用外部工具并处理执行结果的智能系统架构。与传统的被动响应式对话不同，Agent具备主动思考和行动的能力，能够根据任务需求选择合适的工具，观察工具返回的结果，并根据结果调整后续行动，形成一个完整的"思考-行动-观察"循环。Agent架构是实现复杂任务自动化的关键。

## 2 技术背景

大语言模型虽然具备强大的推理和生成能力，但其本身只能输出文本，无法直接执行具体操作。为了让AI能够完成实际任务，需要赋予其使用工具的能力。Agent架构的提出解决了这一问题，它将LLM作为"大脑"，赋予其使用"工具"的能力。ReAct（Reasoning + Acting）范式的出现进一步强化了Agent的推理和行动能力，使其能够处理多步骤的复杂任务。

## 3 核心概念

**Agent（智能体）**：具备自主推理和行动能力的智能程序，能够根据目标自主规划路径并执行操作，而非仅仅响应用户输入。

**工具（Tools）**：Agent可以调用的外部功能模块，如计算器、搜索API、文件读写接口、数据库查询等。每个工具都有明确的输入输出规范。

**记忆（Memory）**：Agent的上下文保持机制，存储历史交互记录，使Agent能够跟踪多轮对话的状态和任务进度。

**提示工程（Prompt Engineering）**：通过精心设计的系统提示来定义Agent的角色、行为规范和输出格式，引导模型按预期方式工作。

**ReAct循环**：Reasoning（推理）→Acting（行动）→Observing（观察）的循环模式，Agent在此循环中不断推进任务执行。

**Plan & Execute（计划执行）**：先规划完整步骤，再按计划逐步执行的模式，适合复杂任务。

**Multi-Agent（多Agent）**：多个Agent协同工作，每个Agent负责特定任务，通过协作完成复杂目标。

## 4 工作原理

Agent系统的工作流程是一个迭代的"思考-行动-观察"循环。在每一轮迭代中，Agent首先分析当前状态和任务目标，进行推理（Reasoning）确定下一步行动；然后选择合适的工具并执行行动（Acting）；最后观察行动结果（Observing），评估是否达到目标。

如果任务未完成，Agent会根据观察到的结果调整策略，进入下一轮循环。这个过程持续进行，直到任务完成或达到最大迭代次数。整个过程中，Agent可以访问工具列表，了解每个工具的功能和参数规范，根据任务需求选择合适的工具。

## 5 技术实现

Agent系统的技术实现主要包含以下几个方面。Agent核心类负责管理任务状态、调用LLM、解析响应、执行工具；工具注册表存储所有可用工具及其描述信息；提示构建器负责构造符合格式要求的提示词，包含系统指令、工具描述、历史记忆等；LLM调用模块负责与AI模型交互；结果解析器从模型输出中提取行动指令。

## 6 Python代码实现与逐行解析

```python
"""
Agent系统架构

Agent架构模式：
- ReAct: Thought-Action-Observation循环
- Plan & Execute: 先规划后执行
- Multi-Agent: 多Agent协作
"""

from typing import Dict, List, Any, Optional
from openai import OpenAI
import json
import os
import re


class TaskExecutionAgent:
    """任务执行Agent
    
    功能：
    - 理解用户任务需求
    - 自主规划执行步骤
    - 调用工具完成任务
    - 处理执行过程中的异常
    """
    
    def __init__(self):
        self.client = OpenAI(api_key=os.getenv("OPENAI_API_KEY"))
        self.memory: List[Dict] = []        # 记忆存储，保存历史推理和观察结果
        self.max_iterations = 10           # 最大迭代次数，防止无限循环
        
        # 注册工具字典，将工具名称映射到具体的工具函数
        self.tools = {
            "calculate": self._tool_calculate,      # 数学计算工具
            "search": self._tool_search,            # 搜索工具（模拟）
            "get_time": self._tool_get_time,        # 获取当前时间
            "file_read": self._tool_file_read,     # 文件读取工具
            "file_write": self._tool_file_write     # 文件写入工具
        }
    
    def _tool_calculate(self, expression: str) -> str:
        """计算器工具：安全地执行数学表达式计算"""
        try:
            # 安全检查：只允许数字和基本运算符
            allowed_chars = set('0123456789+-*/.() ')
            if all(c in allowed_chars for c in expression):
                result = eval(expression)  # 计算表达式结果
                return str(result)
            return "错误：表达式包含非法字符"
        except Exception as e:
            return f"计算错误: {str(e)}"
    
    def _tool_search(self, query: str) -> str:
        """搜索工具（模拟）：返回模拟的搜索结果"""
        return f"搜索结果：找到关于'{query}'的相关信息..."
    
    def _tool_get_time(self, _=None) -> str:
        """获取当前时间：返回当前日期时间"""
        from datetime import datetime
        return datetime.now().strftime("%Y-%m-%d %H:%M:%S")
    
    def _tool_file_read(self, filepath: str) -> str:
        """读取文件工具：从指定路径读取文件内容"""
        try:
            with open(filepath, 'r', encoding='utf-8') as f:
                return f.read()[:1000]  # 限制返回长度
        except Exception as e:
            return f"读取失败: {str(e)}"
    
    def _tool_file_write(self, params: str) -> str:
        """写入文件工具：格式为"文件路径|内容" """
        try:
            parts = params.split('|', 1)  # 使用|分割路径和内容
            if len(parts) != 2:
                return "格式错误，使用: 文件路径|内容"
            filepath, content = parts
            with open(filepath, 'w', encoding='utf-8') as f:
                f.write(content)
            return f"成功写入文件: {filepath}"
        except Exception as e:
            return f"写入失败: {str(e)}"
    
    def _get_system_prompt(self) -> str:
        """获取系统提示：定义Agent的行为规范和工具说明"""
        # 生成所有工具的描述列表
        tool_descriptions = "\n".join([
            f"- {name}: {func.__doc__}" 
            for name, func in self.tools.items()
        ])
        
        return f"""你是一个智能任务执行助手。你可以使用以下工具来完成任务：

{tool_descriptions}

请按以下格式回复：
思考: [你的分析和计划]
行动: [工具名] [参数]
或
完成: [任务结果]

注意：
1. 每次只执行一个工具
2. 观察工具返回后继续下一步
3. 如果任务完成，使用"完成:"格式回复
"""
    
    def execute_task(self, task: str) -> str:
        """执行任务：主入口，启动Agent的任务执行循环"""
        print(f"🎯 任务: {task}\n")
        
        # 清空历史记忆，准备新一轮任务执行
        self.memory = []
        
        # 迭代执行：思考->行动->观察循环
        for iteration in range(self.max_iterations):
            print(f"--- 迭代 {iteration + 1} ---")
            
            # 构建消息：包含系统提示、用户任务和历史记忆
            messages = [
                {"role": "system", "content": self._get_system_prompt()},
                {"role": "user", "content": f"任务: {task}"}
            ]
            
            # 添加历史记忆，帮助Agent了解任务进度
            for mem in self.memory:
                messages.append({"role": "assistant", "content": mem["response"]})
                messages.append({"role": "user", "content": f"观察: {mem['observation']}"})
            
            # 调用LLM获取Agent的决策
            response = self.client.chat.completions.create(
                model="gpt-4o-mini",
                messages=messages,
                temperature=0
            )
            
            content = response.choices[0].message.content
            print(f"🤔 助手:\n{content}\n")
            
            # 检查任务是否完成
            if "完成:" in content:
                result = content.split("完成:", 1)[1].strip()
                print(f"✅ 任务完成: {result}")
                return result
            
            # 解析行动指令，提取工具名和参数
            action_match = re.search(r'行动:\s*(\w+)\s*(.+)?', content, re.DOTALL)
            
            if action_match:
                tool_name = action_match.group(1).strip()
                tool_params = action_match.group(2).strip() if action_match.group(2) else ""
                
                print(f"🔧 执行: {tool_name}({tool_params})")
                
                # 执行工具调用
                if tool_name in self.tools:
                    try:
                        observation = self.tools[tool_name](tool_params)
                    except Exception as e:
                        observation = f"工具执行错误: {str(e)}"
                else:
                    observation = f"未知工具: {tool_name}"
                
                print(f"👁️ 观察: {observation}\n")
                
                # 保存到记忆，供下一轮迭代使用
                self.memory.append({
                    "response": content,
                    "observation": observation
                })
            else:
                print("❌ 无法解析行动，任务中断")
                break
        
        return "任务未完成，达到最大迭代次数"


if __name__ == "__main__":
    agent = TaskExecutionAgent()
    
    # 测试任务
    tasks = [
        "计算 (15 + 25) * 3 / 5 的结果",
    ]
    
    for task in tasks:
        result = agent.execute_task(task)
        print(f"最终结果: {result}\n")
```

def react_architecture():
    """ReAct架构详解"""
    print("""
    ReAct（Reasoning + Acting）是一种结合推理和行动的Agent架构模式。
    它让Agent在执行任务时能够进行显式的推理过程，并在推理指导下选择和执行行动。
    
    核心思想：
    
    1. 推理（Reasoning）
       - 分析当前状态
       - 理解任务目标
       - 规划下一步行动
       - 推理过程可被记录和审视
    
    2. 行动（Acting）
       - 执行选定的工具或操作
       - 获取外部信息或执行任务
       - 产生可观察的结果
    
    3. 观察（Observing）
       - 分析行动结果
       - 更新对任务的理解
       - 决定下一步行动
    
    循环流程：
    
    思考 → 行动 → 观察 → 思考 → 行动 → 观察 → ... → 完成
    
    ReAct的优势：
    
    1. 可解释性
       - 推理过程清晰可见
       - 便于调试和优化
       - 用户可以理解Agent的决策
    
    2. 任务分解
       - 复杂任务分解为简单步骤
       - 每步都可以验证和调整
       - 减少错误累积
    
    3. 错误恢复
       - 能够识别执行错误
       - 根据观察结果调整策略
       - 不容易陷入死循环
    
    提示词模板：
    
    你是一个智能助手。在解决问题时，请按以下格式思考：
    
    思考：<分析当前状况，推理下一步行动>
    行动：<执行的操作>
    观察：<操作结果>
    
    重复这个过程直到问题解决。
    如果问题已解决，请回答"完成: <答案>"。
    
    与传统方法对比：
    
    | 特性       | ReAct         | 纯推理        | 纯行动        |
    |------------|---------------|---------------|---------------|
    | 推理能力   | 强            | 强            | 弱            |
    | 执行能力   | 强            | 弱            | 强            |
    | 可解释性   | 高            | 高            | 低            |
    | 错误恢复   | 支持          | 不支持        | 支持          |
    
    应用场景：
    
    1. 复杂问答
       - 需要多步推理和查询
       - 如：查找信息、计算、总结
    
    2. 任务自动化
       - 多步骤操作
       - 如：文件处理、数据分析
    
    3. 调试和开发
       - 需要理解Agent思维过程
       - 如：代码调试、问题定位
    """)

def plan_execute_architecture():
    """规划-执行架构详解"""
    print("""
    规划-执行（Plan & Execute）架构是一种两阶段的Agent模式。
    第一阶段生成完整的行动计划，第二阶段按计划逐步执行。
    这种架构适合复杂任务的规划和优化。
    
    核心概念：
    
    1. 规划阶段（Plan）
       - 理解任务目标
       - 分析任务需求
       - 生成详细执行计划
       - 可以包含子任务分解
    
    2. 执行阶段（Execute）
       - 按计划顺序执行
       - 监控执行状态
       - 处理执行异常
       - 更新执行进度
    
    执行模式：
    
    1. 顺序执行
       - 按计划步骤顺序执行
       - 简单直观
       - 适合线性依赖的任务
    
    2. 并行执行
       - 识别独立任务
       - 并行执行提高效率
       - 需要处理依赖关系
    
    3. 动态调整
       - 执行中根据结果调整计划
       - 适应变化的情况
       - 更加灵活
    
    架构优势：
    
    1. 全局优化
       - 可以预先优化执行顺序
       - 减少不必要的步骤
       - 提高整体效率
    
    2. 可追溯性
       - 完整的执行计划可见
       - 便于审核和调试
       - 支持计划调整
    
    3. 资源优化
       - 预先分配资源
       - 预测执行时间
       - 优化成本
    
    实现示例：
    
    ```python
    class PlanExecuteAgent:
        def plan(self, task):
            # 分析任务
            # 生成执行计划
            return [
                {"step": 1, "action": "search", "params": {...}},
                {"step": 2, "action": "read", "params": {...}},
                {"step": 3, "action": "analyze", "params": {...}}
            ]
        
        def execute(self, plan):
            results = []
            for step in plan:
                result = self.execute_step(step)
                results.append(result)
                if self.should_adjust(step, result):
                    plan = self.adjust_plan(plan, results)
            return results
    ```
    
    适用场景：
    
    1. 复杂项目任务
       - 多阶段项目
       - 需要规划资源
    
    2. 旅行规划
       - 行程安排
       - 资源预订
    
    3. 数据分析
       - ETL流程
       - 多步骤处理
    
    与ReAct对比：
    
    | 特性         | Plan & Execute | ReAct         |
    |--------------|----------------|---------------|
    | 计划时机     | 执行前         | 执行中        |
    | 适应性       | 低             | 高            |
    | 全局优化     | 支持           | 不支持        |
    | 实现复杂度   | 中             | 低            |
    """)

def multi_agent_collaboration():
    """多Agent协作系统详解"""
    print("""
    多Agent协作系统通过多个专业Agent的协同工作来完成复杂任务。
    每个Agent可以专注于特定领域，通过协作实现更强大的能力。
    
    协作模式：
    
    1. 层次结构
       - 有一个主Agent负责协调
       - 子Agent负责执行具体任务
       - 类似组织架构
    
    2. 平等协作
       - Agent之间平等交互
       - 通过消息传递协作
       - 类似社会网络
    
    3. 专家模式
       - 每个Agent是特定领域专家
       - 按需调用
       - 类似顾问团队
    
    通信机制：
    
    1. 消息传递
       - Agent之间发送消息
       - 包含任务、信息、结果
       - 需要定义消息格式
    
    2. 共享状态
       - 共享工作区/黑板
       - Agent读写共享信息
       - 需要同步机制
    
    3. 工具调用
       - 一个Agent调用其他Agent
       - 类似工具调用
       - 需要注册和发现机制
    
    协作策略：
    
    1. 任务分解
       - 将复杂任务分解为子任务
       - 分配给不同Agent
       - 汇总结果
    
    2. 信息融合
       - 多个Agent提供不同角度
       - 综合形成完整答案
       - 需要冲突解决机制
    
    3. 迭代优化
       - Agent之间互相审查
       - 迭代改进结果
       - 达成共识
    
    实现示例：
    
    ```python
    class MultiAgentSystem:
        def __init__(self):
            self.agents = {
                "researcher": ResearcherAgent(),
                "analyzer": AnalyzerAgent(),
                "writer": WriterAgent()
            }
        
        def collaborate(self, task):
            # 任务分解
            sub_tasks = self.decompose(task)
            
            # 分发给Agent
            results = {}
            for name, sub_task in sub_tasks.items():
                results[name] = self.agents[name].execute(sub_task)
            
            # 融合结果
            final_result = self.fuse(results)
            return final_result
    ```
    
    适用场景：
    
    1. 复杂项目
       - 需要多种技能
       - 大型报告生成
    
    2. 辩论/评审
       - 多角度分析
       - 达成共识
    
    3. 模拟系统
       - 模拟社会行为
       - 角色扮演
    """)

def state_management():
    """状态管理与记忆系统"""
    print("""
    Agent需要维护内部状态来跟踪任务进度、存储记忆、管理上下文。
    良好的状态管理是Agent可靠工作的基础。
    
    状态类型：
    
    1. 执行状态
       - 当前任务进度
       - 已完成步骤
       - 剩余步骤
    
    2. 上下文状态
       - 对话历史
       - 任务背景
       - 用户信息
    
    3. 知识状态
       - 已获取信息
       - 学习到的知识
       - 常识更新
    
    记忆机制：
    
    1. 短期记忆
       - 当前任务相关
       - 存储在内存中
       - 容量有限
    
    2. 长期记忆
       - 持久化存储
       - 经验总结
       - 知识库
    
    3. 工作记忆
       - 当前推理过程
       - 中间结果
       - 临时变量
    
    记忆管理策略：
    
    1. 滑动窗口
       - 保留最近N条记忆
       - 遗忘旧的
    
    2. 重要性加权
       - 评估记忆重要性
       - 优先保留重要记忆
    
    3. 摘要压缩
       - 定期压缩记忆
       - 提取关键信息
    
    4. 检索增强
       - 基于相似度检索
       - 相关记忆召回
    
    实现示例：
    
    ```python
    class MemoryManager:
        def __init__(self, max_tokens=4000):
            self.short_term = []      # 短期记忆
            self.long_term = {}       # 长期记忆
            self.max_tokens = max_tokens
        
        def add(self, memory):
            self.short_term.append(memory)
            self.trim()
        
        def retrieve(self, query):
            # 检索相关记忆
            return self.search(query)
        
        def trim(self):
            # 裁剪超出限制的记忆
            while self.estimate_tokens() > self.max_tokens:
                self.short_term.pop(0)
    ```
    
    状态持久化：
    
    1. 会话级
       - 保存到会话结束
       - Redis/内存
    
    2. 用户级
       - 跨会话保持
       - 数据库存储
    
    3. 永久级
       - 知识积累
       - 向量数据库

def complete_agent_system():
    """完整的Agent系统代码示例"""
    from typing import Dict, List, Any, Callable
    import json
    
    # ========================================
    # 1. 工具基类
    # ========================================
    class BaseTool:
        """工具基类"""
        
        def __init__(self, name: str, description: str):
            self.name = name
            self.description = description
        
        def execute(self, params: Dict) -> str:
            raise NotImplementedError
        
        def get_schema(self) -> Dict:
            """返回工具Schema"""
            return {
                "name": self.name,
                "description": self.description
            }
    
    # ========================================
    # 2. 具体工具实现
    # ========================================
    class CalculatorTool(BaseTool):
        """计算器工具"""
        
        def __init__(self):
            super().__init__("calculator", "执行数学计算")
        
        def execute(self, params: Dict) -> str:
            expression = params.get("expression", "")
            try:
                result = eval(expression)
                return str(result)
            except Exception as e:
                return f"计算错误: {str(e)}"
    
    class SearchTool(BaseTool):
        """搜索工具"""
        
        def __init__(self):
            super().__init__("search", "搜索信息")
        
        def execute(self, params: Dict) -> str:
            query = params.get("query", "")
            # 模拟搜索
            return f"搜索'{query}'的结果：找到相关文档10篇"
    
    class WeatherTool(BaseTool):
        """天气查询工具"""
        
        def __init__(self):
            super().__init__("weather", "查询天气")
        
        def execute(self, params: Dict) -> str:
            city = params.get("city", "北京")
            return f"{city}今天天气：晴，20-28°C"
    
    # ========================================
    # 3. Agent核心
    # ========================================
    class ReActAgent:
        """ReAct架构Agent"""
        
        def __init__(self):
            self.tools: Dict[str, BaseTool] = {}
            self.memory: List[Dict] = []
            self.max_turns = 10
        
        def register_tool(self, tool: BaseTool):
            self.tools[tool.name] = tool
        
        def run(self, task: str) -> str:
            """运行Agent"""
            print(f"🎯 任务: {task}")
            
            for turn in range(self.max_turns):
                print(f"\n--- Turn {turn + 1} ---")
                
                # 1. 推理
                thought = self.think(task)
                print(f"🤔 思考: {thought}")
                
                # 2. 检查是否完成
                if self.is_done(thought):
                    return self.extract_answer(thought)
                
                # 3. 提取行动
                action = self.extract_action(thought)
                if not action:
                    return "无法理解任务"
                
                tool_name, params = action
                print(f"🔧 行动: {tool_name}({params})")
                
                # 4. 执行工具
                if tool_name in self.tools:
                    result = self.tools[tool_name].execute(params)
                else:
                    result = f"未知工具: {tool_name}"
                
                print(f"👁️ 观察: {result}")
                
                # 5. 记录到记忆
                self.memory.append({
                    "thought": thought,
                    "action": f"{tool_name}({params})",
                    "observation": result
                })
            
            return "任务未完成（达到最大轮次）"
        
        def think(self, task: str) -> str:
            """思考 - 这里简化处理，实际调用LLM"""
            # 构建提示
            tool_desc = "\n".join([
                f"- {name}: {tool.description}"
                for name, tool in self.tools.items()
            ])
            
            history = "\n".join([
                f"思考: {m['thought']}\n行动: {m['action']}\n观察: {m['observation']}"
                for m in self.memory
            ])
            
            prompt = f"""任务: {task}

可用工具:
{tool_desc}

历史:
{history}

请按以下格式回复：
思考: <你的分析和计划>
行动: <工具名 参数JSON>
或
完成: <最终答案>"""
            
            # 实际使用中，这里调用LLM
            # return llm.chat(prompt)
            
            # 简化模拟
            if "计算" in task and "3" in task:
                return "行动: calculator {\"expression\": \"15 + 25\"}"
            elif "天气" in task:
                return "行动: weather {\"city\": \"北京\"}"
            else:
                return "完成: 这是一个模拟回答"
        
        def is_done(self, thought: str) -> bool:
            return "完成:" in thought
        
        def extract_answer(self, thought: str) -> str:
            return thought.split("完成:")[1].strip()
        
        def extract_action(self, thought: str) -> tuple:
            import re
            match = re.search(r'行动:\s*(\w+)\s*(\{.+\})?', thought)
            if match:
                tool_name = match.group(1)
                params = json.loads(match.group(2) or "{}")
                return tool_name, params
            return None, {}
    
    # ========================================
    # 4. 运行示例
    # ========================================
    def main():
        # 创建Agent
        agent = ReActAgent()
        
        # 注册工具
        agent.register_tool(CalculatorTool())
        agent.register_tool(SearchTool())
        agent.register_tool(WeatherTool())
        
        # 运行任务
        tasks = [
            "计算15加25的结果",
            "今天北京天气怎么样"
        ]
        
        for task in tasks:
            result = agent.run(task)
            print(f"\n✅ 结果: {result}")
            print("=" * 50)
            
            # 重置记忆
            agent.memory = []

## 7 常见问题

**问题一：Agent输出格式不统一**
部分模型对结构化输出的遵循程度不高，可能不按"思考/行动/完成"格式回复。解决方案是在系统提示中增加更多示例，或使用更可靠的模型。

**问题二：工具参数解析错误**
当参数中包含特殊字符时，正则表达式可能无法正确解析。建议使用更健壮的参数解析方式，或要求模型输出JSON格式的参数。

**问题三：任务陷入无限循环**
Agent可能在某些情况下反复执行相同的动作。可以通过记录已执行的行动序列进行去重，或设置更小的最大迭代次数。

**问题四：工具安全问题**
file_write等工具存在潜在的安全风险，可能被用于写入恶意文件。在生产环境中应添加权限控制和操作审计。

**问题五：上下文长度限制**
长对话可能导致超出模型上下文限制。需要实现智能的上下文管理策略，如摘要压缩或选择性记忆。

**问题六：多Agent通信混乱**
多Agent系统中可能出现通信混乱或死锁。需要设计清晰的通信协议和超时机制。
