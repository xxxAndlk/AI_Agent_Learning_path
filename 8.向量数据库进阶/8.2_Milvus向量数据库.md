# 8.2 Milvus向量数据库

## 1 技术简介

Milvus是一个开源的分布式向量数据库，专为大规模向量检索设计。它支持十亿级向量的存储和毫秒级检索延迟，提供高可用、可扩展的架构，适合企业级生产环境。Milvus由Zilliz公司开发，采用云原生架构，支持Kubernetes部署，提供了丰富的索引类型和查询能力，是目前最受欢迎的向量数据库之一。

## 2 技术背景

随着AI应用的大规模落地，传统的数据库无法满足向量数据的存储和检索需求。向量数据具有高维度、大规模、频繁更新等特点，需要专门的数据库系统来处理。Milvus的出现填补了向量数据库领域的空白，它专门针对向量数据优化，支持水平扩展，能够处理海量向量数据，同时提供了丰富的企业级特性，如高可用、ACID事务支持、数据持久化等。

## 3 核心概念

**集合（Collection）**：Milvus中存储向量数据的基本单元，类似于关系数据库中的表。每个集合有自己的Schema定义，包含字段信息（如id、向量、标量字段）。

**分区（Partition）**：集合内的数据分区，可以将数据按某种规则（如时间、类别）存储在不同的分区中，提高查询效率。

**索引（Index）**：加速向量检索的数据结构，Milvus支持多种索引类型，包括HNSW、IVF_FLAT、IVF_SQ8、IVF_PQ、GPU_INDEX等，不同索引适用于不同的数据规模和精度要求。

**分片（Shard）**：数据在物理存储上的分区方式，Milvus支持自动分片，可以将数据分散存储在多个节点上，实现水平扩展。

**副本（Replica）**：数据的冗余拷贝，用于提高可用性和读取吞吐量。Milvus支持多副本配置，保证服务的高可用性。

## 4 工作原理

Milvus的工作流程分为数据写入和数据查询两个阶段。在数据写入阶段，客户端首先连接到Milvus服务器，然后创建集合和Schema，接着插入向量数据和标量数据。插入的数据会首先写入消息队列，然后由后台程序构建索引，最后持久化到存储系统中。

在数据查询阶段，用户首先将查询向量发送给Milvus服务器，系统根据查询条件选择合适的索引执行搜索。对于带过滤条件的查询（混合搜索），Milvus会先执行标量过滤，然后在过滤结果中进行向量搜索。搜索结果按照相似度排序返回，包含向量ID、距离分数和用户指定的输出字段。

## 5 技术实现

Milvus的技术实现主要包含以下几个方面。首先是连接配置，可以使用URI连接到Milvus服务器，Milvus Lite（嵌入式）使用本地文件，全托管服务使用云端地址。其次是Schema定义，使用FieldSchema定义字段，包括主键字段、标量字段和向量字段，每个字段需要指定数据类型和参数。第三是数据操作，使用collection.insert()插入数据，使用collection.search()执行搜索，使用collection.query()执行标量查询。第四是索引管理，使用collection.create_index()创建索引，需要指定索引类型、距离度量方式和参数。第五是混合搜索，通过expr参数指定过滤表达式，结合向量相似度和标量条件进行查询。

# Milvus使用示例（需要安装: pip install pymilvus）
"""
Milvus架构组件：
1. Milvus Server: 负责查询处理和索引构建
2. Etcd: 元数据存储
3. MinIO/S3: 对象存储（向量数据）
4. Pulsar/Kafka: 消息队列（可选）
"""

def milvus_basic_example():
    """Milvus基础使用示例"""
    try:
        from pymilvus import connections, FieldSchema, CollectionSchema, DataType, Collection, utility
        import numpy as np
        
        # 连接到Milvus服务器
        # 默认连接本地Milvus Lite（无需服务器）
        # 或连接Milvus服务器: connections.connect(host='localhost', port='19530')
        connections.connect(alias="default", uri="./milvus_demo.db")
        
        # 定义集合Schema
        # 1. 定义字段
        fields = [
            FieldSchema(name="id", dtype=DataType.INT64, is_primary=True, auto_id=True),
            FieldSchema(name="text", dtype=DataType.VARCHAR, max_length=512),
            FieldSchema(name="embedding", dtype=DataType.FLOAT_VECTOR, dim=128)
        ]
        
        # 2. 创建Schema
        schema = CollectionSchema(
            fields=fields,
            description="文档向量集合",
            enable_dynamic_field=True  # 允许动态字段
        )
        
        # 3. 创建集合
        collection_name = "document_collection"
        
        # 如果集合已存在，先删除
        if utility.has_collection(collection_name):
            Collection(collection_name).drop()
        
        collection = Collection(name=collection_name, schema=schema)
        print(f"✅ 集合 '{collection_name}' 创建成功")
        
        # 准备数据
        num_entities = 1000
        texts = [f"这是第{i}个文档的内容示例" for i in range(num_entities)]
        embeddings = np.random.random((num_entities, 128)).tolist()
        
        # 插入数据
        entities = [
            texts,
            embeddings
        ]
        
        insert_result = collection.insert(entities)
        print(f"✅ 已插入 {len(insert_result.primary_keys)} 个实体")
        
        # 创建索引（提高搜索效率）
        # IVF_FLAT: 倒排文件索引，平衡速度和精度
        index_params = {
            "metric_type": "L2",           # 距离度量（L2=欧氏距离）
            "index_type": "IVF_FLAT",      # 索引类型
            "params": {"nlist": 128}       # 聚类中心数
        }
        
        collection.create_index(field_name="embedding", index_params=index_params)
        print("✅ 索引创建成功")
        
        # 加载集合到内存（搜索前必须）
        collection.load()
        
        # 向量搜索
        search_params = {
            "metric_type": "L2",
            "params": {"nprobe": 10}        # 搜索时查询的聚类数
        }
        
        # 查询向量
        query_vector = np.random.random((1, 128)).tolist()
        
        results = collection.search(
            data=query_vector,
            anns_field="embedding",
            param=search_params,
            limit=5,                         # 返回最相似的5个
            output_fields=["text"]           # 返回的字段
        )
        
        print("\n搜索结果:")
        for hits in results:
            for hit in hits:
                print(f"ID: {hit.id}, 距离: {hit.distance:.4f}, 文本: {hit.entity.get('text')}")
        
        # 释放集合
        collection.release()
        
        # 删除集合
        # utility.drop_collection(collection_name)
        
    except ImportError:
        print("请先安装Milvus SDK: pip install pymilvus")

def milvus_advanced_features():
    """Milvus高级特性：分区、混合搜索"""
    try:
        from pymilvus import connections, FieldSchema, CollectionSchema, DataType, Collection
        import numpy as np
        
        connections.connect(alias="default", uri="./milvus_advanced.db")
        
        # 定义带标量字段的Schema
        fields = [
            FieldSchema(name="id", dtype=DataType.INT64, is_primary=True, auto_id=True),
            FieldSchema(name="category", dtype=DataType.VARCHAR, max_length=64),
            FieldSchema(name="timestamp", dtype=DataType.INT64),
            FieldSchema(name="embedding", dtype=DataType.FLOAT_VECTOR, dim=64)
        ]
        
        schema = CollectionSchema(fields=fields)
        collection = Collection(name="advanced_collection", schema=schema)
        
        # 创建分区
        # 分区可以隔离数据，提高查询效率
        collection.create_partition(partition_name="tech")
        collection.create_partition(partition_name="science")
        
        # 向特定分区插入数据
        tech_data = [
            ["AI", "ML", "DL"],                                    # category
            [1704067200, 1704153600, 1704240000],                 # timestamp
            np.random.random((3, 64)).tolist()                     # embedding
        ]
        
        collection.insert(tech_data, partition_name="tech")
        print("✅ 数据已插入tech分区")
        
        # 混合搜索：向量相似度 + 标量过滤
        collection.load()
        
        query_vector = np.random.random((1, 64)).tolist()
        
        results = collection.search(
            data=query_vector,
            anns_field="embedding",
            param={"metric_type": "L2", "params": {"nprobe": 16}},
            limit=3,
            expr='category == "AI"',           # 标量过滤条件
            output_fields=["category", "timestamp"]
        )
        
        print("\n混合搜索结果（仅category=AI）:")
        for hits in results:
            for hit in hits:
                print(f"ID: {hit.id}, Category: {hit.entity.get('category')}")
        
        collection.release()
        
    except ImportError:
        print("请先安装Milvus SDK: pip install pymilvus")

def index_types_detail():
    """索引类型详解"""
    print("""
    Milvus支持的索引类型：
    
    1. FLAT（暴力搜索）
       - 无索引，直接计算全部向量距离
       - 100%召回率，但速度最慢
       - 适用于小数据集（<10万）
    
    2. IVF_FLAT（倒排索引）
       - 先聚类，搜索时只计算部分聚类
       - 平衡速度和精度
       - 参数：nlist（聚类数）、nprobe（搜索数）
    
    3. IVF_SQ8（量化索引）
       - 在IVF基础上使用标量量化
       - 减少内存占用，精度略有下降
       - 参数：nlist、nprobe、sq_type
    
    4. IVF_PQ（乘积量化）
       - 在IVF基础上使用乘积量化
       - 大幅压缩存储，高速搜索
       - 参数：nlist、nprobe、m、nbits
    
    5. HNSW（分层小世界图）
       - 基于图的索引
       - 高召回率，高内存占用
       - 参数：M、efConstruction、efSearch
    
    6. GPU索引（需要GPU支持）
       - GPU_IVF_FLAT
       - GPU_IVF_PQ
       - Faiss-GPU索引
    
    索引选择指南：
    
    | 数据规模 | 推荐索引 | 特点 |
    |---------|---------|------|
    | <10万   | FLAT    | 100%召回 |
    | 10万-1千万 | IVF_FLAT | 平衡型 |
    | 千万-1亿 | IVF_PQ   | 高速压缩 |
    | 1亿以上  | HNSW    | 高召回 |
    
    索引参数调优建议：
    
    1. IVF系列
       - nlist: 建议4*sqrt(N)到N/100
       - nprobe: 建议nlist的1%-20%
    
    2. HNSW
       - M: 8-64，越大召回越高
       - efSearch: 32-512，越大召回越高
    """)

def milvus_sdk_advanced():
    """Python SDK高级用法"""
    try:
        from pymilvus import connections, FieldSchema, CollectionSchema, DataType, Collection
        import numpy as np
        
        connections.connect(alias="default", uri="./milvus_sdk_advanced.db")
        
        # 1. 批量操作
        fields = [
            FieldSchema(name="id", dtype=DataType.INT64, is_primary=True, auto_id=True),
            FieldSchema(name="vector", dtype=DataType.FLOAT_VECTOR, dim=128),
            FieldSchema(name="score", dtype=DataType.FLOAT)
        ]
        collection = Collection(name="batch_demo", schema=CollectionSchema(fields=fields))
        
        # 批量插入 - 分批处理大数据
        batch_size = 10000
        total_count = 100000
        
        for i in range(0, total_count, batch_size):
            vectors = np.random.random((batch_size, 128)).tolist()
            scores = np.random.random(batch_size).tolist()
            
            collection.insert([vectors, scores])
            
            if (i + batch_size) % 50000 == 0:
                print(f"已插入 {i + batch_size} 条数据")
        
        print(f"✅ 批量插入完成，共{total_count}条")
        
        # 2. 异步操作
        # 异步插入
        future = collection.insert_async(
            [np.random.random((100, 128)).tolist(), np.random.random(100).tolist()]
        )
        print("异步插入已提交")
        # 等待完成
        future.result()
        print("异步插入完成")
        
        # 3. 事务操作（2.3+版本）
        # with collection.transaction() as t:
        #     collection.insert(...)
        #     collection.delete(...)
        
        # 4. 查询操作 - 标量查询
        collection.load()
        query_results = collection.query(
            expr="score > 0.8",
            limit=10,
            output_fields=["id", "score"]
        )
        print(f"查询到{len(query_results)}条记录")
        
        # 5. 删除操作
        # collection.delete(expr="id in [1, 2, 3]")
        
        # 6. 获取集合统计信息
        print(f"集合统计: {collection.num_entities}")
        
        collection.release()
        
    except ImportError:
        print("请先安装Milvus SDK: pip install pymilvus")

def rag_system_example():
    """完整的企业级RAG系统示例"""
    try:
        from pymilvus import connections, FieldSchema, CollectionSchema, DataType, Collection, utility
        import numpy as np
        
        connections.connect(alias="default", uri="./rag_system.db")
        
        # ========================================
        # 1. 定义文档Schema
        # ========================================
        fields = [
            # 主键
            FieldSchema(name="doc_id", dtype=DataType.INT64, is_primary=True, auto_id=True),
            # 文档元数据
            FieldSchema(name="title", dtype=DataType.VARCHAR, max_length=512),
            FieldSchema(name="content", dtype=DataType.VARCHAR, max_length=65535),
            FieldSchema(name="category", dtype=DataType.VARCHAR, max_length=64),
            FieldSchema(name="author", dtype=DataType.VARCHAR, max_length=128),
            FieldSchema(name="created_at", dtype=DataType.INT64),
            # 向量字段
            FieldSchema(name="embedding", dtype=DataType.FLOAT_VECTOR, dim=768)
        ]
        
        schema = CollectionSchema(
            fields=fields,
            description="企业知识库文档集合",
            enable_dynamic_field=True
        )
        
        # ========================================
        # 2. 创建集合
        # ========================================
        collection_name = "enterprise_docs"
        
        if utility.has_collection(collection_name):
            Collection(collection_name).drop()
        
        collection = Collection(name=collection_name, schema=schema)
        
        # 创建分区
        collection.create_partition("技术文档")
        collection.create_partition("产品文档")
        collection.create_partition("运营文档")
        
        print("✅ RAG系统集合创建完成")
        
        # ========================================
        # 3. 索引配置
        # ========================================
        # 使用HNSW索引（高精度场景）
        index_params = {
            "metric_type": "IP",              # 内积（归一化后等价余弦）
            "index_type": "HNSW",
            "params": {
                "M": 16,                       # 连接数
                "efConstruction": 200          # 构建搜索宽度
            }
        }
        
        collection.create_index("embedding", index_params)
        collection.load()
        
        print("✅ HNSW索引创建完成")
        
        # ========================================
        # 4. 文档入库
        # ========================================
        def embed_text(text: str) -> list:
            """模拟文本向量化 - 实际使用embedding模型"""
            np.random.seed(hash(text) % 2**32)
            return np.random.random(768).tolist()
        
        # 模拟文档数据
        docs = [
            {"title": "产品介绍", "content": "我们的产品是一款智能助手...", "category": "产品文档", "author": "张三"},
            {"title": "技术架构", "content": "系统采用微服务架构...", "category": "技术文档", "author": "李四"},
            {"title": "运营指南", "content": "如何有效运营产品...", "category": "运营文档", "author": "王五"},
        ]
        
        for doc in docs:
            vectors = [embed_text(doc["content"])]
            import time
            collection.insert([
                [doc["title"]],
                [doc["content"]],
                [doc["category"]],
                [doc["author"]],
                [int(time.time())],
                vectors
            ])
        
        print("✅ 文档入库完成")
        
        # ========================================
        # 5. RAG检索
        # ========================================
        def rag_retrieve(query: str, top_k: int = 3):
            """RAG检索函数"""
            # 向量化查询
            query_vector = [embed_text(query)]
            
            # 混合搜索：向量相似度 + 标量过滤
            results = collection.search(
                data=query_vector,
                anns_field="embedding",
                param={"metric_type": "IP", "params": {"efSearch": 64}},
                limit=top_k,
                expr='category == "技术文档"',    # 可选过滤
                output_fields=["title", "content", "category", "author"]
            )
            
            # 整理结果
            retrieved_docs = []
            for hits in results:
                for hit in hits:
                    retrieved_docs.append({
                        "doc_id": hit.id,
                        "title": hit.entity.get("title"),
                        "content": hit.entity.get("content"),
                        "category": hit.entity.get("category"),
                        "score": hit.score
                    })
            
            return retrieved_docs
        
        # 测试检索
        query = "系统架构相关文档"
        results = rag_retrieve(query)
        
        print(f"\n查询: {query}")
        print(f"检索到 {len(results)} 条结果:")
        for i, doc in enumerate(results, 1):
            print(f"\n[{i}] {doc['title']} (相似度: {doc['score']:.4f})")
            print(f"    分类: {doc['category']}")
        
        collection.release()
        
    except ImportError:
        print("请先安装Milvus SDK: pip install pymilvus")


if __name__ == "__main__":

## 7 常见问题

**问题一：Milvus服务无法启动**
Milvus需要多个组件协同工作（etcd、minio等），使用docker-compose部署时需要确保所有容器正常运行。检查端口占用、磁盘空间、网络配置等因素。

**问题二：搜索结果召回率低**
可能原因包括：索引参数不合理、数据未加载到内存、查询参数设置不当等。可以通过调整索引参数（如HNSW的efSearch、IVF的nprobe）来提高召回率。

**问题三：混合搜索性能差**
标量过滤会显著影响搜索性能，因为需要先过滤再向量搜索。优化建议包括：减少过滤条件、使用预分区、合理设计索引。

**问题四：数据插入慢**
Milvus支持批量插入，但单次插入数据量过大时会把内存。合理设置批量大小（建议1000-10000条），分批插入可以提高性能。

**问题五：内存占用过高**
HNSW等图索引内存占用较高。可以通过减小M参数、使用IVF-PQ索引、或限制加载到内存的数据量来解决。

**问题六：集群部署复杂度高**
生产环境建议使用Kubernetes部署，可以参考官方Helm Chart。需要提前规划好资源容量和 高可用配置。

## 8 应用场景

**大规模语义搜索**：存储 billions 级别的文档向量，支持毫秒级语义检索，适用于企业搜索、电商搜索等场景。

**推荐系统**：将用户和物品表示为向量，使用向量相似度匹配实现个性化推荐，支持实时推荐需求。

**图像/视频检索**：存储图像和视频的特征向量，支持以图搜图、相似视频检索等应用。

**RAG企业级部署**：作为企业知识库的向量存储，支持高并发、多租户、安全合规等企业级需求。

**AI应用基础设施**：作为AI应用的后端数据库，为聊天机器人、智能客服、RAG系统等提供向量存储和检索能力。

**欺诈检测**：存储交易行为的向量表示，通过相似度匹配识别欺诈模式。

**药物发现**：存储分子结构的向量表示，支持相似药物分子检索。

**异常检测**：将系统行为表示为向量，检测异常行为模式。
