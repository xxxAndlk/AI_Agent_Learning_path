# 10.4 向量索引算法

## 1 技术简介

向量索引算法是向量数据库的核心技术，用于在高维向量空间中快速找到最近邻。由于精确搜索在高维空间中的计算复杂度极高，近似最近邻（ANN）算法通过牺牲少量精度换取大幅度的性能提升。理解不同索引算法的原理和适用场景，对于选择合适的向量数据库和优化检索性能至关重要。

## 2 技术背景

随着深度学习的发展，向量嵌入技术被广泛应用于各种AI场景。然而，高维向量的精确搜索面临"维度诅咒"问题——搜索复杂度随维度指数增长。假设有100万个128维的向量，使用暴力搜索需要计算100万次距离并排序，这在实时应用中是根本无法接受的。ANN算法的出现解决了这一难题，它通过预先构建索引结构，大幅减少搜索时需要计算的向量数量，使得毫秒级搜索成为可能。

## 3 核心概念

**近似最近邻（ANN）**：与精确最近邻（ENN）相对，ANN允许返回不完全精确的结果，但搜索速度大幅提升。召回率是衡量ANN算法质量的关键指标。

**聚类（Clustering）**：将向量空间划分为多个区域，每个区域有一个中心点（质心）。搜索时只需查找与查询向量最近的几个聚类。

**量化（Quantization）**：将高维向量压缩为低维编码，减少存储空间和计算量。乘积量化（PQ）是典型的量化方法。

**图索引（Graph Index）**：通过图结构组织向量，利用图的导航性质快速定位最近邻。HNSW是图索引的代表算法。

**索引参数**：不同索引有不同的参数需要调优，如IVF的nlist（聚类数）、nprobe（搜索的聚类数），HNSW的M（连接数）、efSearch（搜索宽度），PQ的m（子向量数）等。

## 4 工作原理

向量索引的工作原理因算法类型而异。Flat索引（暴力搜索）不构建任何索引结构，直接计算查询向量与所有数据库向量的距离，返回最近的K个。这是最精确的方法，但复杂度为O(N)，适合小规模数据。

IVF（倒排文件）索引通过聚类将向量空间划分为多个区域。构建时使用K-means将向量聚类成nlist个簇；搜索时首先找到最近的nprobe个聚类中心，然后只在这些聚类中搜索。这将复杂度降低到O(N/nlist * nprobe)。

HNSW（分层导航小世界）索引构建一个分层图结构。顶层稀疏、下层稠密，搜索时从顶层开始，通过图链接逐步向下导航，最终在底层找到最近邻。这是一种启发式搜索，复杂度接近O(log N)。

PQ（乘积量化）将高维向量分割为多个子向量，每个子向量独立进行K-means聚类。存储时只保存聚类中心ID（编码），搜索时通过查表快速计算近似距离。这大大降低了存储和计算成本，但引入了量化误差。

## 5 技术实现

向量索引的实现主要包含索引构建和索引搜索两个过程。在FAISS中，可以使用不同的索引类来实现各种算法：IndexFlatL2用于精确搜索；IndexIVFFlat用于倒排文件索引；IndexHNSWFlat用于图索引；IndexPQ用于乘积量化；IndexIVFPQ是IVF和PQ的组合。

参数调优是获得最佳性能的关键。对于IVF索引，nlist决定了聚类数量，通常设置为4 * sqrt(N)到N/100；nprobe决定了搜索的聚类数，增大可以提高召回率但降低速度。对于HNSW索引，M决定了每个节点的连接数，越大召回率越高但内存占用越大；efSearch决定了搜索时的候选集大小，越大召回率越高但速度越慢。对于PQ索引，m是子向量数，nbits是编码位数，这两个参数决定了压缩率和精度。

## 6 Python代码实现与逐行解析

```python
import faiss
import numpy as np
from typing import List, Tuple
import time

def hnsw_example():
    """HNSW索引示例
    
    HNSW (Hierarchical Navigable Small World) 是目前最先进的ANN算法之一
    适合大规模、高召回率的场景
    """
    print("=" * 60)
    print("HNSW索引示例")
    print("=" * 60)
    
    # 参数设置
    d = 128                             # 向量维度
    nb = 100000                         # 数据库向量数量
    nq = 1000                           # 查询向量数量
    k = 10                              # 返回最近邻数量
    
    print(f"\n数据集参数:")
    print(f"  维度: {d}")
    print(f"  数据库大小: {nb}")
    print(f"  查询数量: {nq}")
    print(f"  返回Top-K: {k}")
    
    # 生成随机数据
    np.random.seed(42)
    xb = np.random.random((nb, d)).astype('float32')
    xq = np.random.random((nq, d)).astype('float32')
    
    # 归一化向量（用于余弦相似度）
    faiss.normalize_L2(xb)
    faiss.normalize_L2(xq)
    
    # ============ 创建HNSW索引 ============
    print("\n--- 创建HNSW索引 ---")
    
    # HNSW参数
    M = 16                              # 每个节点的连接数，越大召回率越高，内存占用越大
    efConstruction = 200                # 构建时的搜索范围，越大构建越慢，质量越好
    
    # 创建索引
    # IndexHNSWFlat: 使用Flat存储的HNSW索引
    index = faiss.IndexHNSWFlat(d, M)
    index.hnsw.efConstruction = efConstruction
    
    print(f"HNSW参数:")
    print(f"  M (连接数): {M}")
    print(f"  efConstruction (构建搜索范围): {efConstruction}")
    
    # 添加向量
    print("\n添加向量到索引...")
    start_time = time.time()
    index.add(xb)
    build_time = time.time() - start_time
    print(f"✅ 索引构建完成，耗时: {build_time:.2f}秒")
    print(f"  索引大小: {nb} 个向量")
    
    # ============ 搜索 ============
    print("\n--- 执行搜索 ---")
    
    # 设置搜索参数
    efSearch = 64                       # 搜索时的候选集大小，越大召回率越高，速度越慢
    index.hnsw.efSearch = efSearch
    
    print(f"搜索参数 efSearch: {efSearch}")
    
    # 执行搜索
    start_time = time.time()
    D, I = index.search(xq, k)
    search_time = time.time() - start_time
    
    print(f"\n搜索结果:")
    print(f"  查询次数: {nq}")
    print(f"  总耗时: {search_time:.4f}秒")
    print(f"  平均单次查询: {(search_time/nq)*1000:.2f}ms")
    print(f"  QPS: {nq/search_time:.1f}")
    
    print(f"\n查询示例 (前3个):")
    for i in range(min(3, nq)):
        print(f"  查询{i}: 最近邻索引 {I[i][:5]}, 距离 {D[i][:5]}")
    
    # ============ 与暴力搜索对比 ============
    print("\n--- 精度验证 (与暴力搜索对比) ---")
    
    # 创建Flat索引作为基准
    index_flat = faiss.IndexFlatIP(d)   # 内积距离（归一化后等价于余弦相似度）
    index_flat.add(xb)
    
    # 暴力搜索
    D_gt, I_gt = index_flat.search(xq, k)
    
    # 计算召回率
    recalls = []
    for i in range(nq):
        # 计算HNSW结果中有多少在真实Top-K中
        correct = len(set(I[i]) & set(I_gt[i]))
        recalls.append(correct / k)
    
    avg_recall = np.mean(recalls)
    print(f"召回率 (Recall@{k}): {avg_recall*100:.2f}%")
    print(f"  平均每个查询找到 {avg_recall*k:.1f}/{k} 个真实最近邻")
    
    # ============ 参数调优演示 ============
    print("\n--- efSearch参数影响 ---")
    ef_values = [16, 32, 64, 128, 256]
    
    print(f"{'efSearch':<10} {'Recall@10':<12} {'Query Time(ms)':<15} {'QPS':<10}")
    print("-" * 50)
    
    for ef in ef_values:
        index.hnsw.efSearch = ef
        
        # 搜索
        start = time.time()
        D_test, I_test = index.search(xq, k)
        elapsed = time.time() - start
        
        # 计算召回率
        recalls_test = [len(set(I_test[i]) & set(I_gt[i])) / k for i in range(nq)]
        recall = np.mean(recalls_test)
        
        print(f"{ef:<10} {recall*100:>6.2f}%      {(elapsed/nq)*1000:>6.2f}          {nq/elapsed:>6.1f}")


def hnsw_memory_usage():
    """分析HNSW内存占用"""
    print("\n" + "=" * 60)
    print("HNSW内存占用分析")
    print("=" * 60)
    
    d = 128
    nb = 100000
    M_values = [8, 16, 32, 64]
    
    print(f"\n向量维度: {d}, 向量数量: {nb}")
    print(f"\n{'M':<5} {'内存占用(MB)':<15} {'每向量(bytes)':<15}")
    print("-" * 40)
    
    for M in M_values:
        index = faiss.IndexHNSWFlat(d, M)
        
        # 生成数据并添加
        xb = np.random.random((nb, d)).astype('float32')
        index.add(xb)
        
        # 估算内存（简化计算）
        # 原始向量 + HNSW图结构
        vector_memory = nb * d * 4  # float32 = 4 bytes
        graph_memory = nb * M * 2 * 4  # 每个连接存储邻居ID（int32）
        total_memory = (vector_memory + graph_memory) / (1024 * 1024)
        
        bytes_per_vector = (vector_memory + graph_memory) / nb
        
        print(f"{M:<5} {total_memory:>8.2f}        {bytes_per_vector:>8.1f}")
    
    print("\n注意:")
    print("  - M越大，召回率越高，但内存占用越大")
    print("  - M=16是常用的平衡点")


if __name__ == "__main__":
    hnsw_example()
    hnsw_memory_usage()
```

def brute_force_search():
    """暴力搜索与精确算法详解"""
    print("""
    暴力搜索（Flat Index）是最简单也是最精确的向量检索方法。
    它不构建任何索引结构，直接计算查询向量与数据库中所有向量的距离，
    然后返回最近的K个向量。
    
    算法原理：
    
    1. 距离计算
       - 欧氏距离 (L2): sqrt(sum((xi-yi)^2))
       - 余弦相似度: (x·y) / (||x|| * ||y||)
       - 内积 (IP): x·y
    
    2. 排序
       - 计算所有距离后进行排序
       - 可以使用堆排序优化Top-K查找
    
    3. 返回结果
       - 返回距离最小的K个向量
    
    复杂度分析：
    - 时间复杂度: O(N * D)，N是向量数量，D是维度
    - 空间复杂度: O(N * D)
    
    适用场景：
    - 数据量较小（<10万）
    - 需要100%召回率
    - 作为其他算法的基准
    
    FAISS实现：
    
    # L2距离
    index = faiss.IndexFlatL2(d)
    index.add(xb)
    D, I = index.search(xq, k)
    
    # 内积（归一化后等价于余弦相似度）
    faiss.normalize_L2(xb)
    index = faiss.IndexFlatIP(d)
    index.add(xb)
    D, I = index.search(xq, k)
    
    优化技巧：
    
    1. 批量搜索
       - 一次搜索多个查询向量
       - 利用SIMD指令并行计算
    
    2. GPU加速
       - 使用GPU版本的FAISS
       - 适合大规模数据
    
    3. 预计算距离矩阵
       - 静态数据集可以预计算
       - 动态数据不适用
    """)

def inverted_index_ivf():
    """倒排索引（IVF）详解"""
    print("""
    IVF（Inverted File）倒排索引是一种基于聚类的索引算法。
    它的核心思想是将向量空间划分为多个聚类（簇），
    搜索时只需要在少数几个最相关的聚类中查找。
    
    算法原理：
    
    1. 索引构建阶段
       - 使用K-means算法将N个向量聚类为nlist个簇
       - 每个簇有一个质心（centroid）
       - 每个向量归属于某个簇，并记录与质心的距离
    
    2. 搜索阶段
       - 计算查询向量与所有质心的距离
       - 找到最近的nprobe个簇
       - 在这些簇中搜索最近邻
    
    复杂度分析：
    - 索引构建: O(N * D * iter)
    - 搜索: O(N/nlist * nprobe * D)
    - 空间: O(nlist * D + N * D)
    
    参数调优：
    
    1. nlist（聚类数量）
       - 建议值: 4 * sqrt(N) 到 N/100
       - 越大构建越慢，搜索越快
       - 典型值: 100-10000
    
    2. nprobe（搜索的聚类数）
       - 建议值: nlist的1%-20%
       - 越大召回率越高，速度越慢
       - 典型值: 1-50
    
    IVF变体：
    
    1. IVF_FLAT
       - 簇内使用精确搜索
       - 精度高，速度较快
       - 内存占用中等
    
    2. IVF_SQ8
       - 簇内使用标量量化
       - 压缩存储，精度略有下降
       - 内存占用小
    
    3. IVF_PQ
       - 簇内使用乘积量化
       - 高度压缩，极快搜索
       - 精度下降明显
    
    FAISS代码示例：
    
    # 创建IVF_FLAT索引
    quantizer = faiss.IndexFlatL2(d)        # 质心搜索使用Flat
    index = faiss.IndexIVFFlat(quantizer, d, nlist)
    
    index.train(xb)                          # 训练聚类
    index.add(xb)                           # 添加向量
    
    # 搜索
    index.nprobe = nprobe                   # 设置搜索的聚类数
    D, I = index.search(xq, k)
    
    召回率与性能：
    
    | nprobe/nlist | 召回率 | 速度提升 |
    |--------------|--------|----------|
    | 1%           | ~70%   | 100x     |
    | 5%           | ~90%   | 20x      |
    | 10%          | ~95%   | 10x      |
    | 20%          | ~98%   | 5x       |
    """)

def hnsw_detailed_principle():
    """HNSW详细原理"""
    print("""
    HNSW（Hierarchical Navigable Small World）是一种基于图结构的近似最近邻搜索算法。
    它受到小世界网络（Small World）理论的启发，结合了跳表（Skip List）的分层思想，
    实现了高效的高维向量检索。
    
    核心思想：
    
    1. 分层图结构
       - 顶层：最稀疏，节点少，连接远
       - 底层：最稠密，节点多，连接近
       - 每层形成一个导航图
    
    2. 搜索过程
       - 从最高层开始，找到距离最近的节点
       - 沿着该节点的边下降到下一层
       - 重复直到最底层
       - 在最底层进行精细搜索
    
    3. 边的构建
       - 每个节点连接到最近的M个邻居
       - 使用启发式方法保证图的小世界性质
    
    数据结构：
    
    1. 跳表层
       - 0层：包含所有向量
       - i+1层：i层的一个子集
       - 概率衰减：P ≈ 1/M
    
    2. 邻接表
       - 每个节点维护一个邻居列表
       - 存储邻居ID和到邻居的距离
    
    算法步骤：
    
    1. 构建阶段
       - 为每个向量分配随机层数
       - 从最高层开始插入
       - 在每层找到最近的M个邻居
       - 建立连接
    
    2. 搜索阶段
       - 从最高层开始
       - 贪心搜索找到当前层的最近邻
       - 下降到下一层
       - 重复直到第0层
       - 使用优先队列维护候选集
    
    3. 插入阶段
       - 类似搜索，找到插入位置
       - 在各层建立新节点的连接
       - 更新现有节点的连接
    
    参数详解：
    
    1. M（每个节点的连接数）
       - 影响图的质量和内存
       - 太小：图不连通
       - 太大：内存占用高
       - 推荐：8-64
    
    2. efConstruction（构建时的候选集大小）
       - 影响索引质量
       - 越大构建越慢，质量越好
       - 推荐：100-300
    
    3. efSearch（搜索时的候选集大小）
       - 影响搜索召回率和速度
       - 越大召回越高，速度越慢
       - 推荐：32-512
    
    时间复杂度：
    - 构建：O(N * M * logN)
    - 搜索：O(logN * M)
    
    空间复杂度：
    - O(N * M * D)  # 图结构
    - O(N * D)      # 原始向量
    
    优点：
    - 高召回率（>95%）
    - 搜索速度快
    - 支持增量插入
    
    缺点：
    - 内存占用高
    - 构建时间长
    - 参数调优复杂
    
    与IVF对比：
    
    | 特性        | HNSW       | IVF_FLAT   |
    |-------------|------------|------------|
    | 召回率      | 高(>95%)   | 中(80-95%) |
    | 搜索速度    | 快         | 中         |
    | 内存占用    | 高         | 中         |
    | 构建时间    | 长         | 短         |
    | 增量插入    | 支持       | 困难       |
    """)

def product_quantization():
    """乘积量化（PQ）详解"""
    print("""
    乘积量化（Product Quantization，PQ）是一种向量压缩技术。
    它将高维向量分割成多个子向量，对每个子向量独立进行量化，
    从而实现高压缩率。
    
    核心思想：
    
    1. 分割
       - 将D维向量分成m个D/m维的子向量
       - 例如：128维分成8个16维子向量
    
    2. 独立量化
       - 对每个子向量空间独立进行K-means聚类
       - 每个子向量用聚类中心ID表示（编码）
       - 每个中心存储实际向量
    
    3. 距离计算
       - 搜索时计算查询向量与编码的距离
       - 使用查表法加速
    
    量化过程：
    
    1. 学习码本
       - 在训练数据上运行K-means
       - 为每个子空间生成k个聚类中心
       - k通常设为256（8bit）
    
    2. 编码
       - 对于每个子向量，找到最近的中心
       - 用中心ID作为编码
       - 原始向量 → m个编码
    
    3. 存储
       - 只存储编码（m个int8）
       - 存储码本（m * k * D/m个float）
    
    压缩率：
    
    原始：N * D * 4 bytes
    编码：N * m * 1 byte
    码本：m * k * D/m * 4 byte
    
    例子：N=1M, D=128, m=8, k=256
    原始：512 MB
    编码：8 MB
    码本：32 KB
    总计：约8 MB（压缩64倍）
    
    距离计算：
    
    1. 分段距离
       - 查询向量也分成m个子向量
       - 对每个子向量，计算到所有中心的距离
       - 查表得到预计算的距离
    
    2. 合并距离
       - 将m个子向量距离相加
       - 得到近似的全向量距离
    
    参数调优：
    
    1. m（子向量数）
       - 影响压缩率和精度
       - 越多压缩率越高，误差越大
       - 典型值：4-16
    
    2. nbits（编码位数）
       - 决定每个子空间的中心数
       - k = 2^nbits
       - 典型值：8（256个中心）
    
    3. nlist（与IVF结合时）
       - IVF_PQ中的聚类数
       - 先聚类再量化
    
    PQ变体：
    
    1. OPQ（优化乘积量化）
       - 旋转向量以优化量化效果
       - 更高的精度
    
    2. CVP（成分向量量化）
       - 对不同成分使用不同精度
       - 重要成分用更多bit
    
    3. IVF_PQ
       - IVF + PQ组合
       - 先聚类，再对簇内PQ
       - 平衡速度和精度
    
    FAISS代码示例：
    
    # 简单PQ
    m = 8                              # 子向量数
    nbits = 8                          # 编码位数
    index = faiss.IndexPQ(d, m, nbits)
    index.train(xb)
    index.add(xb)
    D, I = index.search(xq, k)
    
    # IVF_PQ
    nlist = 100
    quantizer = faiss.IndexFlatL2(d)
    index = faiss.IndexIVFPQ(quantizer, d, nlist, m, nbits)
    index.train(xb)
    index.add(xb)
    index.nprobe = 10
    D, I = index.search(xq, k)
    """)

    1. 聚类阶段
       - 使用K-means将数据分成nlist个簇
       - 每个簇有一个质心

       - 对每个簇内的向量使用PQ量化
       - 独立的码本
    
    3. 搜索阶段
       - 找到最近的nprobe个簇
       - 在每个簇内使用PQ搜索
    
    性能特点：
    
    1. 速度
       - 非常快，适合大规模数据
       - 比IVF_FLAT快10-100倍

       - 适合内存受限场景
    
    参数选择：
    
    1. nlist（聚类数）

       - 典型值：5-50

def complete_index_comparison():
    """完整索引对比示例"""
    import faiss
    import numpy as np
    import time
    
    # 测试配置
    d = 128              # 维度
    nb = 50000           # 数据库大小
    nq = 100             # 查询数量
    k = 10               # Top-K
    
    # 生成测试数据
    np.random.seed(42)
    xb = np.random.random((nb, d)).astype('float32')
    xq = np.random.random((nq, d)).astype('float32')
    
    # 归一化（用于内积）
    faiss.normalize_L2(xb)
    faiss.normalize_L2(xq)
    
    results = []
    
    # 1. Flat索引
    print("\n--- Flat索引 ---")
    index_flat = faiss.IndexFlatIP(d)
    index_flat.add(xb)
    
    start = time.time()
    D_flat, I_flat = index_flat.search(xq, k)
    time_flat = time.time() - start
    results.append(("Flat", 100.0, time_flat))
    print(f"耗时: {time_flat*1000:.2f}ms")
    
    # 2. IVF_FLAT索引
    print("\n--- IVF_FLAT索引 ---")
    nlist = 100
    quantizer = faiss.IndexFlatIP(d)
    index_ivf = faiss.IndexIVFFlat(quantizer, d, nlist)
    index_ivf.train(xb)
    index_ivf.add(xb)
    index_ivf.nprobe = 10
    
    start = time.time()
    D_ivf, I_ivf = index_ivf.search(xq, k)
    time_ivf = time.time() - start
    
    recall_ivf = np.mean([len(set(I_ivf[i]) & set(I_flat[i]))/k for i in range(nq)]) * 100
    results.append(("IVF_FLAT", recall_ivf, time_ivf))
    print(f"耗时: {time_ivf*1000:.2f}ms, 召回率: {recall_ivf:.1f}%")
    
    # 3. IVF_PQ索引
    print("\n--- IVF_PQ索引 ---")
    m = 8
    nbits = 8
    index_pq = faiss.IndexIVFPQ(quantizer, d, nlist, m, nbits)
    index_pq.train(xb)
    index_pq.add(xb)
    index_pq.nprobe = 10
    
    start = time.time()
    D_pq, I_pq = index_pq.search(xq, k)
    time_pq = time.time() - start
    
    recall_pq = np.mean([len(set(I_pq[i]) & set(I_flat[i]))/k for i in range(nq)]) * 100
    results.append(("IVF_PQ", recall_pq, time_pq))
    print(f"耗时: {time_pq*1000:.2f}ms, 召回率: {recall_pq:.1f}%")
    
    # 4. HNSW索引
    print("\n--- HNSW索引 ---")
    M = 16
    efSearch = 32
    index_hnsw = faiss.IndexHNSWFlat(d, M)
    index_hnsw.hnsw.efSearch = efSearch
    index_hnsw.add(xb)
    
    start = time.time()
    D_hnsw, I_hnsw = index_hnsw.search(xq, k)
    time_hnsw = time.time() - start
    
    recall_hnsw = np.mean([len(set(I_hnsw[i]) & set(I_flat[i]))/k for i in range(nq)]) * 100
    results.append(("HNSW", recall_hnsw, time_hnsw))
    print(f"耗时: {time_hnsw*1000:.2f}ms, 召回率: {recall_hnsw:.1f}%")
    
    # 汇总
    print("\n" + "="*50)
    print("性能对比汇总")
    print("="*50)
    print(f"{'索引类型':<12} {'召回率':<10} {'耗时(ms)':<10} {'相对速度':<10}")
    print("-"*50)
    for name, recall, t in results:
        speedup = results[0][2] / t
        print(f"{name:<12} {recall:>6.1f}%    {t*1000:>6.2f}     {speedup:>6.1f}x")

## 7 常见问题

**问题一：索引构建时间过长**
HNSW的efConstruction参数越大，构建时间越长。如果构建时间成为瓶颈，可以适当减小efConstruction（但会影响召回率），或者使用IVF系列索引。

**问题二：召回率不理想**
首先确认是否正确设置了efSearch参数（需要足够大）；其次检查M参数是否合适（对召回率影响很大）；最后确认向量是否正确归一化（如果使用内积作为度量）。

**问题三：内存占用过高**
HNSW索引的内存占用与向量数量和M参数成正比。可以通过减小M、使用IVF索引、或使用PQ压缩来降低内存占用。

**问题四：IVF索引搜索慢**
确保在搜索前调用index.nprobe设置合理的搜索聚类数；nprobe越大搜索越慢但召回率越高；通常设置为nlist的5%-20%效果较好。

**问题五：PQ索引精度下降明显**
PQ量化会引入量化误差，导致精度下降。可以通过增加nlist（配合IVF_PQ）、增大m（子向量数）或使用OPQ来改善。

**问题六：向量化维度选择**
维度越高，存储和计算成本越高。可以通过PCA降维来平衡精度和性能。通常128-512维是常见的嵌入维度。

## 8 应用场景

**高精度检索场景**：如语义搜索、问答系统，对召回率要求高，可以接受较高的延迟和内存占用，推荐使用HNSW索引。

**内存受限场景**：如边缘设备、嵌入式应用，需要控制内存占用，推荐使用IVF-PQ或PQ索引。

**大规模数据场景**：如亿级向量检索，需要分布式架构和高吞吐量，推荐使用Milvus的分布式索引。

**延迟敏感场景**：如实时推荐、在线搜索，需要毫秒级响应，推荐使用HNSW并调优参数。

**平衡场景**：如一般的企业搜索，数据量在百万级，推荐使用IVF_FLAT或HNSW，根据具体需求调整参数。

**图像视频检索**：高维特征向量检索，需要特殊处理，推荐使用降维+HNSW或IVF_PQ组合方案。

**推荐系统实时召回**：大规模物品召回，需要高吞吐，推荐使用IVF_PQ多路召回。

**基因序列检索**：生物信息学场景，向量维度极高，推荐使用降维+IVF索引组合。
